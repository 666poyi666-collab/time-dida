# FocusLink 「极光剧场」设计语言 v0.3

> 本文档定义 FocusLink v0.3 系列的全新视觉语言。
> 上一代「Notion 纸面」设计被完全替换，目标是建立截然不同的舞台式专注体验。

---

## 一、设计理念：为什么推倒重来

v0.2.x 系列采用「左右分栏 + Notion 纸面」布局，存在三个根本问题：

1. **左右分栏割裂注意力**：计时与任务并排，视线在两栏间反复横跳，破坏专注的沉浸感。
2. **视觉单调平庸**：纸面背景 + 蓝色按钮，与任意办公工具雷同，没有「专注剧场」的仪式感。
3. **小窗像残缺的主窗**：小窗只是主窗的缩小版，缺乏独立的设计语言。

「极光剧场」用三个核心决定回应：

- **舞台优先**：计时器是唯一的视觉主角，占据中央舞台；任务退为可召唤的抽屉，不常驻。
- **深邃画布**：以深色极光画布为基底，用环境光与辉光制造空间纵深，而非平面色块。
- **字体性格**：用 Sora（几何展示）+ JetBrains Mono（精密数字）+ Manrope（正文）替代通用 Inter，赋予技术感与精致感。

## 二、布局架构：从「左右分栏」到「舞台 + 抽屉」

### 主窗口骨架

```
┌─────────────────────────────────────────────────────┐
│  侧轨 │              极光舞台                       │
│       │                                           │
│  计时 │         ╭─────────────╮                   │
│  历史 │         │   弧光环     │                   │
│  设置 │         │  00:32:14   │   ← 巨型计时舞台  │
│       │         ╰─────────────╯                   │
│       │      [当前任务 · 控制]                     │
│       │                                           │
│       │  ▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭▭  ← 水平时间码头       │
│  窗口 │                                           │
└───────┴───────────────────────────────────────────┘
```

- **左侧轨**（56px）：垂直导航，图标 + 状态指示，取代顶部水平 nav。
- **中央舞台**：计时器、控制、统计的沉浸区。
- **水平时间码头**：片段时间线从竖向改为水平条带，贴底。
- **任务抽屉**：从右侧滑入的覆盖层，不常驻、召唤式出现。

### 小窗骨架

```
  收起态                  展开态
┌──────────┐    ┌────────────────────────┐
│ ● 专注中  │    │ ● 专注中 │ 任务标题    │
│ 32:14     │    │  00:32:14   累计 1:02  │
│ 累计 1:02 │    │  [⏸ 暂停]  [⏹ 结束]    │
└──────────┘    └────────────────────────┘
```

- 收起态：极简双行，时间即主角。
- 展开态：玻璃浮卡，时间 + 累计 + 控制。
- 视觉语言独立于主窗，但共享令牌。

## 三、色彩系统：深邃画布 + 极光辉光

### 深色模式（签名外观）

| 令牌 | 值 | 用途 |
|------|-----|------|
| `--app-bg` | `9 11 16` (#090b10) | 画布底色，近黑深蓝 |
| `--app-surface` | `16 20 28` (#10141c) | 卡片表面 |
| `--app-surface-2` | `24 30 40` (#181e28) | 次级表面 |
| `--app-elevated` | `32 40 52` (#202834) | 抬升层 |
| `--app-accent` | `129 140 248` (#818cf8) | 靛蓝辉光主色 |
| `--app-success` | `52 211 153` (#34d399) | 专注态辉光 |
| `--app-warning` | `251 146 60` (#fb923c) | 暂停态辉光 |

画布不再是纯色，而是「极光环境光」：多层径向渐变模拟极光环境，专注时画布泛起冷绿辉光，暂停时泛起暖橙辉光。

### 浅色模式

保持纸面质感但更冷调克制，避免与 v0.2 的暖纸面混淆。

### 主题色变体

保留 6 个主题色，但全部调整为更高饱和度的「辉光色」，在深色画布上有发光感。

## 四、字体系统：技术感 + 精致感

| 用途 | 字体 | 特征 |
|------|------|------|
| 计时数字 | JetBrains Mono | 等宽、精密、技术感，`tnum` + `zero` |
| 展示标题 | Sora 600/700 | 几何、现代、有性格 |
| 正文 Latin | Manrope 400/500/600 | 圆润精致、可读性强 |
| 正文中文 | Microsoft YaHei UI | 系统中文字体 |

**禁用** Inter / Roboto / Arial 等通用字体作为主字体。

---

## 五、圆角与纵深

- 圆角加大：卡片 `20px`，按钮 `14px`，小窗 `24px`，营造更柔软的「悬浮」感。
- 纵深分层：背景极光 → 表面卡片 → 抬升控件，三层 `box-shadow` 制造空间感。
- 辉光语义：专注态 `success` 外发光，暂停态 `warning` 外发光，取代纯边框。

---

## 六、动效原则

沿用 v0.2 的动效节奏（120/180/260ms），但增加：

- **舞台呼吸**：计时舞台在 running 时有极弱的辉光呼吸（3.2s 循环）。
- **抽屉滑入**：任务抽屉从右侧滑入，带 scrim 遮罩淡入。
- **弧光流转**：弧光环的进度沿圆周流转，专注态每秒推进。
- **状态变色**：专注↔暂停时，画布环境光做 260ms 色彩渐变。

---

## 七、状态标签规范（不可违反）

沿用 AGENTS.md：
- `已关联` / `未关联` 仅用于本地任务关联。
- `已同步` / `未同步` / `同步失败` 仅用于云同步队列状态。
- dida CLI 同步描述为 `同步到滴答清单`。
