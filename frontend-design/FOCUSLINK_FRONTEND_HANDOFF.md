# FocusLink 前端/UI 交接文件

> 目标读者：接手 UI 视觉与前端体验设计的 AI 或设计师。
> 当前版本：v0.2.28。此文件只描述真实 `src/` 前端，不引用未接入应用的概念设计目录。

## 1. 产品定位

FocusLink 是一个桌面效率工具，不是聊天产品，也不是营销网站。

核心功能：
- 专注计时器。
- 专注片段与暂停片段记录。
- dida / 滴答清单任务关联。
- 历史记录按时间范围查看。
- 专注小窗用于后台工作时快速查看状态。

视觉方向：
- 冷静、干净、现代、长期使用不累眼。
- 浅色优先，深色不能坏。
- 主色可保留薄荷绿 / emerald。
- 避免花哨背景、过度渐变、大面积单一紫蓝、营销式 hero。
- 需要像成熟桌面工具，不要像网页半成品。

## 2. 当前代码入口

前端实现目录：
- `src/App.tsx`：主窗口布局、顶部导航、左右栏分割线。
- `src/components/TimerPanel.tsx`：左侧专注计时区。
- `src/components/TaskPanel.tsx`：右侧 dida / 滴答清单任务树。
- `src/components/HistoryPanel.tsx`：历史记录与 session 展开详情。
- `src/components/SettingsPanel.tsx`：设置页。
- `src/components/MiniWindow.tsx`：专注小窗。
- `src/index.css`：全局 tokens、字体、卡片、按钮、动效、小窗壳层。
- `shared/miniWindowLayout.ts`：小窗固定尺寸策略。

后端和 IPC 不应由 UI 设计任务随意改动：
- `electron/`：主进程、计时、SQLite、dida CLI、同步。
- `shared/types.ts`：前后端共享数据模型。

## 3. 主界面结构不能推翻

真实主界面是左右双栏：

左侧专注计时区：
- 顶部状态。
- 继续 / 暂停 / 结束按钮。
- 快捷键提示。
- 片段时间线。
- 专注片段与暂停片段列表。
- 当前片段高亮。
- 任务关联入口。

右侧 dida 任务区：
- dida CLI 连接状态。
- 清单选择器。
- 总任务 / 当前可见 / 专注关联统计。
- 搜索框。
- 已完成任务隐藏提示。
- 任务树列表。
- 同步队列状态。

设计时不要改成仪表盘首页，不要改成 AI chat，不要改成营销页。

## 4. 小窗设计要求

小窗当前明确只有两种用户状态：

缩小态：
- 固定尺寸 `260×88`。
- 只显示两组数据：
  - 专注中：当前专注、累计专注。
  - 暂停中：当前暂停、累计暂停。
- 只保留展开入口，不塞入复杂操作。
- 不能出现内容挤压、文字重叠、棱角露底。

展开态：
- 固定尺寸 `420×184`。
- 显示：
  - 状态。
  - 当前任务。
  - 当前片段大计时。
  - 当前专注。
  - 累计专注。
  - 当前暂停。
  - 累计暂停。
  - 总历时。
  - 暂停 / 继续 / 开始、结束、打开主窗口、缩小。
- 布局必须稳定，不能依赖拖拽到某个奇怪尺寸才看得清。

重要实现约束：
- 小窗尺寸策略在 `shared/miniWindowLayout.ts`。
- 主进程通过 `mini:collapse` / `mini:expand` 直接 setBounds。
- 不要用主进程逐帧动画改窗口大小，容易导致窗口跳动。
- 小窗页面 `html.mini-window-page` 背景必须透明，否则圆角会显示成方角。

## 5. 历史记录设计要求

历史记录需要像时间账本，而不是杂乱卡片堆。

时间筛选：
- 今天。
- 昨天。
- 近 7 天。
- 半个月。
- 1 个月。
- 自定义时间。

Session 展开后优先级：
1. 先显示专注片段与暂停片段混合时间线。
2. 本地 / 云端状态只能作为小状态 chip，不要占大块。
3. 批量关联、默认任务、同步操作放在次级区域。

视觉规则：
- 专注片段使用绿色。
- 暂停片段使用橙色。
- 当前片段需要清晰高亮。
- 云端同步状态要有用，但不能喧宾夺主。

## 6. 任务区设计要求

任务树不是普通列表。

必须保留：
- 父任务默认折叠。
- 搜索命中子任务时自动展开父任务。
- 清空搜索后恢复搜索前折叠状态。
- 已完成任务默认隐藏。
- 子任务缩进清楚。
- 当前关联任务标记。
- 本次默认任务标记。
- 同步队列状态。

不要做：
- 不要用奇怪箭头污染叶子任务标题。
- 不要把任务做成过大的营销卡片。
- 不要牺牲列表密度。

## 7. 视觉 tokens

当前 tokens 在 `src/index.css`。

核心方向：
- 字体：`"Microsoft YaHei UI", "Inter", "Segoe UI", system-ui, sans-serif`
- 数字：tabular nums，避免计时跳动。
- 主色：emerald / mint。
- 暂停：warning orange。
- 卡片：边框主导、轻阴影。
- 圆角：主界面卡片约 16-18px，小窗可 22-26px。
- 背景：稳定灰阶，不要粗糙纹理。

v0.2.28 当前落地风格：
- 主壳使用 `app-shell` / `topbar-shell` / `workspace-pane`，避免页面像浏览器半成品。
- 顶部导航使用 `segmented-nav`，当前 tab 以 Notion 蓝浅底和细边框提示。
- 全局背景使用 Notion 式暖纸面 `#f6f5f4`，结构主色为蓝，专注语义使用绿色，暂停语义使用橙色。
- 应用图标、托盘图标和主窗口 `BrandMark` 共享同一套“纸面卡片 + 焦点环 + 任务节点”母题。
- 左右分割线使用 `pane-divider`，默认细线，hover / dragging 只轻微显色。
- 任务树使用 `task-row` / `task-row-parent` / `task-row-child` / `task-tree-children`。
- 小窗使用 `mini-window-shell`，固定两态：260×88 与 420×184。
- 计时区主按钮必须按状态使用语义色：开始/继续/新专注为绿色，暂停为橙色，结构操作才用蓝色。
- `finished` 状态的主按钮必须先 reset 再开始新专注，不能调用无效 toggle。
- 历史折叠行必须用片段摘要判断关联状态，不允许把“默认任务未设置”显示成“未关联”。
- 同步状态只使用“已同步 / 未同步 / 同步失败”等确定词，不使用“可同步”。
- 保留轻量 motion 工具类，但不要再堆毛玻璃、大面积发光和滑动切页。

## 8. 禁止改动的底层逻辑

UI 任务不要改：
- Timer 状态机。
- SQLite schema。
- dida CLI Provider。
- 快捷键注册逻辑。
- IPC 通道协议。
- Session / Segment / PauseEvent 数据模型。
- 任务关联逻辑。
- 历史记录关联逻辑。

可以改：
- React 组件布局。
- Tailwind class。
- `src/index.css` tokens。
- 小型纯 UI helper。
- 文案与信息层级。

## 9. 验收清单

每次 UI 修改至少验证：
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run dist`

重点人工验收：
- 主窗口左右栏不被推翻。
- 小窗缩小态只显示当前 + 累计。
- 小窗展开态信息完整且不重叠。
- 自启动不会弹主界面。
- 计时数字仍实时跳动。
- dida 任务仍能显示、搜索、筛选、关联。
- 历史记录展开后先看到片段与暂停记录。
