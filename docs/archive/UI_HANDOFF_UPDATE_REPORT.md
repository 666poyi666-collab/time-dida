# UI 重构交接文档更新（v0.1.5）

> 本轮更新 `docs/UI_REDESIGN_BRIEF.md`，为 UI AI 接手界面重做补充 v0.1.5 关键约束。

---

## 一、更新动机

v0.1.5 对交互流程做了关键调整，UI AI 在重做界面前必须知晓：

1. 小窗 UI 不合格，贴边收纳已禁用，需要 P0 重做
2. 任务选择流程已改为「先选任务再开始」
3. TaskPicker 默认折叠是硬要求
4. Segment 与任务的绑定关系不可破坏
5. Timer 状态机不可重写

---

## 二、UI_REDESIGN_BRIEF.md 更新内容

### 2.1 新增「〇、v0.1.5 重要更新（必读）」章节

在文档顶部新增 4 个小节，作为 UI AI 的强制约束：

#### 0.1 小窗 UI 当前不合格，列为 P0 重做对象

- 专注小窗必须重做视觉
- 贴边自动收纳 v0.1.5 已默认关闭（乱跳/乱缩放/挡屏幕）
- **UI AI 不要沿用现有贴边收纳实现**，需重新设计贴边交互
- 保留的交互入口：手动收起/展开、托盘控制、主窗口隐藏时自动显示、专注开始时主窗口不在前台则自动显示
- 收起/展开动画请在渲染层用 CSS/transform 实现，**不要在主进程做 incremental setBounds 动画**

#### 0.2 任务选择流程已改为「先选任务，再开始」

- idle 状态可预选任务
- 主进程新增 `timer:start-with-task` 原子启动接口
- TimerPanel 已实现预选 UI
- **UI AI 重做 TimerPanel 时必须保留预选流程与 `startWithTask` 调用**
- 不选任务直接开始仍允许

#### 0.3 TaskPicker / TaskTree 默认折叠是硬要求

- 所有任务树组件必须默认折叠父任务
- 统一规则：默认折叠 → 点击展开 → 搜索展开 → 清空恢复
- 已完成默认隐藏
- **重做视觉时必须保留这套折叠逻辑**

#### 0.4 不可破坏的业务契约

- 不要破坏任务关联逻辑（Segment ↔ Task 绑定）
- 不要破坏 Segment 与任务的绑定关系
- 不要重写 Timer 状态机

### 2.2 更新「二、当前核心功能状态」

#### 2.1 已可用列表

- 专注小窗项补充：贴边自动收纳已默认关闭
- 新增：所有任务树/TaskPicker 默认折叠父任务（v0.1.5 硬要求）
- 新增：idle 状态预选任务 + startWithTask 原子启动
- 新增：暂停后继续新建 Segment 并继承 Session 默认任务
- 新增：历史记录后补关联 + 批量关联

#### 2.2 绝对不能动

- IPC 接口契约补充：v0.1.5 新增 `timer:start-with-task`（原子启动）

### 2.3 更新「三、需要重做的界面」优先级表

- **专注小窗** 提升为 **P0 极高**，说明「不要沿用现有贴边实现」
- 任务区补充：默认折叠是硬要求（v0.1.5）
- TimerPanel 补充：已有 idle 预选任务流程，保留
- 设置页补充：贴边收纳开关已禁用

### 2.4 更新「八、专注小窗设计建议」

章节顶部新增 v0.1.5 重点提示块：
- 小窗列为 P0 重做对象
- 贴边收纳不要沿用现有实现
- 保留的交互入口清单
- 动画实现约束（渲染层 CSS，不要主进程 setBounds 动画）

### 2.5 更新「8.5 小窗必须支持的交互」

- 手动收起/展开：v0.1.5 已改为无动画直接 setBounds
- 贴边自动收纳：⚠️ 已默认关闭，UI AI 需重新设计
- 鼠标悬停展开：⚠️ 当前未实现，UI AI 可在渲染层添加
- 新增：主窗口隐藏到托盘时自动显示小窗（保留）
- 新增：专注开始时主窗口不在前台则自动显示小窗（保留）

### 2.6 更新「十七、优先级建议」

P0 顺序调整：
1. 小窗视觉重做（v0.1.5 重点，不要沿用现有贴边实现）
2. 字体替换
3. 主窗口计时区视觉优化（保留 idle 预选任务流程 + startWithTask 调用）
4. 任务树视觉优化（保留逻辑，默认折叠是硬要求）

### 2.7 版本号更新

- 当前版本：0.1.2 → 0.1.5
