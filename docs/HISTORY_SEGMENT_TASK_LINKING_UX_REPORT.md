# 历史记录片段任务关联 UX 报告

> 版本：v0.1.8
> Commit：`1ed0760ad2aa73a53d018a6d4ddc2f4c762a131e`
> 日期：2026-06-30

## 一、为什么默认只强调专注片段

大部分情况下，只有专注片段需要关联任务。暂停片段通常不需要关联任务——暂停是休息或中断，不产生有效工作时间。

因此历史记录的视觉优先级：
- **专注片段**：上方、重点展示、默认提供关联入口
- **暂停记录**：下方、弱化显示、默认不提供关联入口

## 二、历史记录布局

一个 Session 展开后分为两块：

```
┌─────────────────────────────────────┐
│ 时间统计（专注时长 / 暂停时长 / 总历时）│
├─────────────────────────────────────┤
│ 任务时间分布                          │
├─────────────────────────────────────┤
│ 同步可见性                            │
├─────────────────────────────────────┤
│ 本次专注默认任务 + 批量操作            │
├─────────────────────────────────────┤
│ 🟢 专注片段 (N)          ← 上方重点   │
│   专注片段 1                          │
│   时间：15:48 → 15:49                │
│   时长：0:26                          │
│   任务：未关联 / 任务名               │
│   [关联任务] [更换任务] [清除关联]     │
├─────────────────────────────────────┤
│ 🟠 暂停记录 (M)          ← 下方弱化   │
│   暂停 1 · 15:49 → 15:50 · 0:18  ⋮  │
│   暂停 2 · 15:51 → 15:52 · 0:24  ⋮  │
└─────────────────────────────────────┘
```

## 三、专注片段展示

每个专注片段显示：
- 编号 + 开始时间 → 结束时间
- 持续时长
- 任务关联状态（已关联 / 未关联）
- 操作按钮：
  - 已关联：`[更换任务]` `[清除关联]` `[完成任务]`
  - 未关联：`[关联任务]`（主按钮，醒目）

未关联片段用虚线橙色边框 + 淡橙色背景提示。

## 四、暂停片段如何通过三个点选择关联任务

### 交互流程

1. 暂停记录默认只显示：编号 + 时间区间 + 时长 + `⋮`（MoreVertical）按钮
2. 点击 `⋮` 弹出菜单：
   - 「关联到任务」
   - 「添加备注」
3. 点击「关联到任务」→ 弹出 Toast 提示

### 为什么弹出 Toast 而非 TaskPicker

当前 PauseEvent 表结构没有 `taskId` 字段。本轮硬约束「不要改 SQLite 既有字段含义」，因此暂停片段关联任务功能需要后续版本扩展本地数据结构后才能启用。

Toast 文案：
```
暂停片段暂不支持关联任务，该能力将在后续版本扩展本地数据结构后启用。
```

这保证了：
- UX 结构完整（三点菜单 + 关联选项存在）
- 不违反 schema 约束
- 对用户诚实（明确告知是后续功能）

## 五、云端同步为什么暂时后置

**云端写入 dida 专注记录：后续任务，当前版本不做。**

原因：
1. dida CLI 可能没有安全的「追加专注记录」接口
2. 之前 `update --content` 可能是覆盖式，有数据风险
3. 云端写入需要单独设计安全机制
4. 本轮不能为了云端同步破坏本地记录

当前版本只保证本地数据库正确保存：
- Session
- Focus Segment
- Pause Segment
- Segment 对应任务
- 历史记录关联状态
