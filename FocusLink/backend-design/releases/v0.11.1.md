## 本次更新

### 完整动态材质工作面

- 承接 v0.11.0 已完成但未形成 GitHub Release 的全部界面更新：低对比环境光场、轨道轮廓、稀疏粒子、状态呼吸光和强调色伴生色共同构成连续工作面。
- 专注、任务、统计与设置统一使用高不透明材质面、折射边缘和定向阴影；按钮、面板、数字与状态动画使用分层 transform / opacity，启用“减少动态效果”时会停用持续动画。
- 默认字体升级为 Manrope Variable + Noto Sans SC Variable 的“澄澈”方案，同时保留 Geist“精准”方案并支持即时切换；六种强调色同时驱动按钮、导航与环境层。

### 任务工作台与双同步

- 任务页直接使用滴答清单，支持搜索、强制刷新、完成、恢复、6 秒撤销，以及按最近完成、名称和截止日期整理已完成任务；不再暴露多余的“任务来源”交互。
- 滴答 CLI 保持评论优先、任务内容后备、稳定 segment marker 幂等和 checklist 父任务更新；本地关联状态与云同步队列状态继续分开表达。
- 番茄 To-do 手动同步支持在标准安装路径按需建立桥接，后台重试不会擅自启动或结束外部应用；未识别标题继续归入“学习”。

### 过渡式边缘小窗

- 小窗严格只有折叠 `184×35` 与展开 `256×92` 两种固定尺寸；折叠态突出 25px 当前时间和 3px 真实专注进度轨，展开态以 31px 时间为视觉核心。
- 展开态在单一紧凑布局中集中显示任务、当前时间、累计专注、累计暂停、总历时和全部当前控制，不使用嵌套卡片。
- 原生 Windows move loop 在释放后执行“吸附 → 可见过渡 → 自动折叠”，离开边缘自动恢复；过渡期间重新拖动会取消待折叠。

## 修复

- GitHub Windows runner 会把 frameless 窗口的 `window.outerHeight` 混入不可见系统边框；小窗 smoke 现在使用 viewport、填满 viewport 的 shell 与 PNG 像素共同验证真实内容尺寸，outer size 只保留作诊断。
- 临时 user-data 清理加入 Windows 有界重试，避免 Electron 日志尾写入造成的 `ENOTEMPTY` 掩盖首个错误。
- 主窗与小窗 smoke 从 `package.json` 自动推导 `release-v*`，不再硬编码旧版本目录。
- 已公开的 `v0.11.0` tag 在 GitHub Release 创建前被 runner 特有的错误断言阻断；该 tag 保持不可变，本补丁版本从新的干净源码提交重新打包和复验。

## 升级提示

- 用户数据库、任务关联、计时记录和同步队列无需迁移。
- 旧设置缺少 `fontProfile` 时自动使用“澄澈”方案，可在设置 > 体验切回 Geist“精准”。
- 安装版与便携版均包含同一 `0.11.1 / 4088dba` 构建；隔离安装验证不会关闭或覆盖当前运行中的旧版会话。

## 已知限制

- 番茄 To-do 当前只能确认 `cloudSyncUploadRecord` 返回 success；客户端没有 PCRecord 的独立云端回读与远端删除 API，因此不会把本地 marker 或 `isSynced=1` 描述成“云端回读通过”。
- 本机小窗实测覆盖单显示器四边；多显示器与混合 DPI 由纯布局几何测试覆盖。
- 无已知阻断问题。
