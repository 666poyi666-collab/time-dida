# FocusLink 前端设计规范

> 状态：v0.11.x 单一真相（当前实现 v0.11.4）
>
> 产品边界：桌面专注计时、任务执行、统计账本与同步控制；不是聊天应用、营销页或通用仪表盘。

## 1. 设计目标

FocusLink 的界面要像一件高频桌面工具：信息密度高、反馈快、层级少、状态一眼可辨。v0.11 以中性画布、稳定对比、可切换内置字体和单一动态环境层为基础；精致来自排版、间距、折射边缘、材质纹理、受控状态光与有层级的动效，不来自反复套卡片、大面积 blur、文字 glow 或高频装饰动画。

设计优先级固定为：

1. 计时和任务操作准确。
2. 当前状态、关联状态和同步状态不混淆。
3. 高频路径一步到位，低频设置按需展开。
4. 深浅主题都舒适、清晰且有足够对比度，不用高亮白或近黑大面积刺激视线。
5. 动效用于解释状态变化和空间关系，快而自然，并服从 reduced-motion。

## 2. 信息架构

主窗口只有四个同级入口，顺序固定：

| 入口 | 核心内容 | 禁止内容 |
| --- | --- | --- |
| 专注 | 当前任务、计时、开始/暂停/继续/结束、会话片段 | 完整任务管理、重复统计大卡 |
| 任务 | 滴答任务树、搜索、清单、排序、完成/恢复、关联并开始 | 暴露“任务来源”、用设置页代替任务操作、点选后再多一次无意义确认 |
| 统计 | 时间范围、专注/暂停/总历时、历史会话、关联与同步诊断 | 卡片瀑布流、模糊的“可同步”状态 |
| 设置 | 连接、同步、体验与版本信息 | 首屏暴露原始模板、数据库路径和冗长诊断 |

顶栏只出现品牌、四个入口和必要的窗口操作。页面内部不再重复同名大标题；任务抽屉只能作为“选择任务”的短流程，不能替代独立任务页。

### 专注页

- 空闲时以时间和主要操作为唯一视觉中心。
- 产生片段后，计时器与本次片段账本必须组成一个连续工作台；左右区域可以用分隔线区分，但不得再套两个相互竞争的独立大卡。当前片段以轻量状态色带和窄状态边表达，不使用第三层圆角方框。
- 计时核心以大字号时间建立绝对主层级，状态与片段编号在同一基线上辅助阅读；背景只允许局部流动光面和低对比纹理，底部细活动轨表达“正在记录”而不伪装成有终点的进度。禁止圆轨、交叉轴线、跳动均衡器和文字 glow。
- 选中的任务与当前片段任务必须可辨；没有上下文时不显示“更换、清除、设为默认”等按钮。
- 开始/继续使用品牌强调色作为唯一主操作；绿色只表达正在专注、完成和同步成功，暂停使用红色状态语义，结束是次级危险操作。
- 有片段后才展开紧凑的混合时间线。专注、暂停和总历时使用统一三时间模型，不由 renderer 自行推算持久化事实。

### 任务页

- 任务页的产品对象固定是“滴答清单”；不提供本地 / CLI / OAuth 来源切换。CLI 优先、OAuth 后备是连接策略，只在设置和诊断中说明。
- 首屏只请求活动任务。用户进入“已完成”后才请求历史，可选近 30 / 90 / 365 天，默认 90 天；切换时旧响应不得覆盖新请求。
- 待完成可按滴答顺序、截止日期或任务名称排序；已完成可按 `completedAt` 最近完成、任务名称或截止日期排序。完成时间缺失时稳定放到已知时间之后。
- 每行至少包含完成状态、标题、清单/层级上下文和主操作。完成与取消完成使用同一个可逆控件。
- 完成后该行先显示状态变更，并在 6 秒撤销条内允许一键恢复；错过撤销后仍可在已完成列表中找到并恢复，不得形成“点错后消失”的死路。
- 任务树和完成历史首批最多渲染 120 项，用户显式选择“再显示”后每次再增加 120 项；搜索、筛选和排序变更后回到首批。
- 行内主要操作是“关联”或“关联并开始”；更多操作只在确有需要时出现。
- 父任务默认折叠；搜索命中子项时临时展开父链。清空搜索后恢复用户的折叠状态。
- checklist 子项的父任务解析属于后端，前端只传稳定任务身份，不伪造普通任务。
- 乐观更新失败时必须回滚并提示真实错误；不得把 CLI 的 `undefined` 当成功。

### 统计页

- 默认近 7 天，筛选与摘要保持在一个连续工具面上。
- 摘要下使用一块连续分析画布：按日专注使用柱、趋势线、面积和刻度组成的组合图作为主视图，专注/暂停构成与单次专注排行是同画布内的辅助视图。三者共享标题、摘要、分隔、色彩和刻度语言，不拆成三张圆角后台卡片；图表随时间范围重算，只有一个有效日期时也必须保留上方余量，不得让单柱冲顶；没有记录时显示空状态，不得用假数据填图。
- 会话行可展开查看专注/暂停混合时间线、任务、学科及两个同步域。
- `已关联 / 未关联` 只表示本地任务关联；`已同步 / 未同步 / 同步失败` 只表示滴答同步。
- 番茄 To-do 使用“已写入本地 / 待上传 / 上传已确认”等独立文案；不把上传接口 success 写成独立云端回读或远端可删除。
- 短于一分钟的有效记录显示 `<1 分钟`，不能显示误导性的 `0 分钟`。
- 展开会话详情必须用 request id 与当前展开 id 双重核对；连续点击不同会话时，较慢的旧 IPC 响应不得把详情写到新行。加载失败在当前行内可见且可重试，不锁死整页。
- 计时每秒 tick 只能更新会话 id / 状态等统计页真正使用的原子值，不得订阅完整 `TimerSnapshot` 导致全页列表重渲染。

### 设置页

- **连接**：滴答 CLI 探测、OAuth 后备连接与诊断；标题不使用“任务来源”。
- **同步**：滴答同步模式与队列、番茄 To-do 学科与待上传重试。手动重试可在客户端未运行时显示“正在连接番茄 To-do”并走按需桥接；若已以普通模式运行且无桥，必须明确提示“请完全退出番茄 To-do 后再连接”，不显示为自动重启或后台处理中。
- 后台周期重试不会启动外部应用；界面不得在没有用户手动操作时暗示番茄 To-do 将被打开。
- **体验**：主题、快捷键、小窗、启动行为和版本。
- 正常状态只展示是否就绪；可执行文件、模板、数据库路径和原始输出统一放入高级折叠区。

## 3. 主题与颜色 token

保留暗色主题作为选项，默认为精密明亮主题。亮色使用带微绿的中性浅灰画布和瓷白表面，不用大面积高亮纯白；暗色使用石墨灰层级，不用无层次近黑底色。颜色对比由文字、边界和表面的明度差建立，不靠荧光色和外发光补救。

### 必备语义 token

| 代码 token | 用途 | 约束 |
| --- | --- | --- |
| `--app-bg` | 应用画布 | 与 surface 有稳定明度差 |
| `--app-surface` / `--app-surface-2` / `--app-surface-3` / `--app-elevated` | 主工作面、次级面与浮层 | 不用阴影代替全部层级 |
| `--app-text` / `--app-muted` / `--app-subtle` | 三级文字 | 正文对背景至少 4.5:1 |
| `--app-border` / `--app-border-strong` | 分隔与焦点边界 | 不能在亮色主题中消失 |
| `--app-accent` / `--app-accent-hover` | 品牌、选中导航、主要操作和键盘焦点 | 不承担专注或暂停状态 |
| `--app-success` | 正在专注、专注数据、任务完成与同步成功 | 绿色；不代替品牌主操作 |
| `--app-pause` | 暂停状态 | 红色；不复用 warning |
| `--app-warning` | 限流、待处理、提示 | 橙色 |
| `--app-danger` | 删除等破坏操作 | 不与普通“结束”混成同一层级 |

亮色基准可参考画布 `245 247 244`、表面 `254 255 252`和正文 `20 27 23`；暗色基准可参考画布 `20 23 22`、表面 `27 31 29`和正文 `232 237 233`。具体值以 CSS token 为代码真值。所有状态还要用图标、文字或形状区分，不能只靠颜色。

### 材质与光效

- 主窗只允许一个位于内容背后的环境层：低对比主/伴生色光场、两条轨道轮廓、稀疏确定性粒子和低频扫光。它不承载信息、不拦截指针，在任务/统计/设置页自动降低存在感。
- 主工作面使用高不透明表面、1px 折射边缘、微纹理和中性定向阴影；大型工作面与顶栏禁止 `backdrop-filter`，避免多层亚克力和 Electron 合成黑块。浮层才可使用单层短时模糊。
- 状态色可进入环境层和短边缘高光，但文字与计时数字不得 glow；亮度必须低于内容面，不能损害对比或形成整屏霓虹。
- 所有光影不得改变可点击区域、造成文字对比下降或越过透明小窗边界。

## 4. 空间与排版

- 采用 4px 基线；常用间距为 4/8/12/16/24/32。
- 高频任务行高度建议 40–48px；按钮触控目标不小于 32×32px。
- 主窗口默认与最小尺寸以 `shared/mainWindowLayout.ts` 为代码真值。
- 默认“舒展”字体以 `Manrope Variable` 负责拉丁、标题和数字，并以 `Noto Sans SC Variable` 负责中文回退；“锐界”字体使用 `Geist Variable`，中文优先回退到 `Microsoft YaHei UI`，同时使用更高字重和更紧字距。两者通过 `fontProfile` 持久化并可在设置 > 体验即时切换，正文与计时数字的 computed font family / weight / tracking 必须可区分；计时启用 `tabular-nums`，`JetBrains Mono` 仅用于诊断输出和代码片段。
- 主窗普通正文建议 13–14px，任务标题不小于 12.5px，所有可读内容和可操作文案不小于 11px。10px 仅可用于不承载唯一信息的 badge/辅助标签；主窗 9px 以下文字不得出现。固定小窗的非核心微标签是唯一例外。
- 同级元素的圆角、描边和阴影必须来自 token；不得在组件中散落任意值。

## 5. 动效与点击反馈

动效节奏固定为三档，数值以共享 CSS token 为真值：

| 类型 | v0.11 基准 | 使用场景 |
| --- | --- | --- |
| 快速 | 140ms | 按下、hover、行内操作 |
| 标准 | 220–340ms | 页面方向过渡、浮层、状态切换 |
| 空间 | 320ms | 小窗展开、较大空间变化 |

- 点击必须有按下、完成或失败反馈；主要按钮采用轻微 scale/亮度反馈，不能移动布局。
- 页面切换使用约 340ms 的 8–14px 短位移、轻微缩放与 opacity 收敛，禁止 blur/filter；方向与导航顺序一致，退出层必须立即失去 pointer events。
- 列表插入、完成和取消完成保持位置连续；被完成项淡出或移动前先呈现状态改变。
- 弹窗打开时焦点进入弹窗，关闭后回到触发器；Escape 关闭，Tab 不穿透。
- 常规动效只使用 `transform`、`opacity`、颜色和边界色，不动画 blur、大阴影或会触发全页布局的属性。弹性参数只能用于单一小控件且不得有明显过冲；持续动画仅允许环境层、加载旋转和运行状态指示，环境周期不得短于 9 秒。
- `prefers-reduced-motion` 下取消位移、呼吸与弹性变化，只保留必要的短淡入和状态结果。

## 6. 小窗

小窗只有两种固定状态，尺寸常量以 `shared/miniWindowLayout.ts` 为唯一代码真值；CSS、Electron bounds、设置与 smoke 测试不得各写一份数字。v0.11 保持收起 `184×35`、展开 `256×92`，重建比例、进度轨与贴边过渡；修改代码常量时必须在同一变更中更新本文。

### 收起态：边缘进度条

- 比当前展开态显著更小，是单行或窄条形进度组件。
- 只显示进度/状态、当前时间和展开入口，不展示任务详情、累计统计或额外按钮；这是收起态的完整产品契约。
- 当前时间使用 25px 与等宽数字，是小窗唯一主视觉层级；底部 3px 进度轨表达有效专注占总历时比例，状态标识不得挤压时间可读宽度。
- 拖到当前显示器左/右/上/下任一边缘后自动收起；靠边方向决定圆角和展开方向。
- 不在用户拖动过程中抢占鼠标。释放后先完成吸附，再播放 320ms 收束/淡缩提示，最后改变为折叠尺寸；过渡中重新拖动必须取消折叠。
- 点击或向屏幕内侧拖动可展开；专注/暂停状态变化不应无故把窗口移离边缘。

### 展开态：紧凑控制台

- 大小约等于旧收起卡片的量级，全部信息集中在一个有秩序的密集布局中。
- 一屏同时容纳当前任务、状态/时间、累计专注/暂停/总历时以及开始/暂停/继续/结束和收起入口。
- 使用对齐网格和分隔，而不是多层卡片；当前时间使用 31px，是第一视觉层级，控制位于稳定区域，任何状态切换都不应跳位。
- 从边缘展开时向屏幕内部生长，始终通过当前显示器 work area 校正，不越界。

### 吸附参数与防抖

| 参数 | v0.11 验收值 | 作用 |
| --- | --- | --- |
| 进入阈值 | 14px | 释放时进入边缘吸附区 |
| 离开阈值 | 30px | 大于进入阈值，避免临界位置抖动 |
| 释放后稳定等待 | 260ms | 收到 Windows 原生拖拽释放事件后再开始吸附；按住不动时绝不改尺寸 |
| 贴边收束过渡 | 320ms | 吸附后保持展开尺寸并给出可见反馈，再折叠 |
| 拖离展开延迟 | 140ms | 完全离开所有边缘后自动展开 |
| 点击展开防回弹 | 900ms | 点击箭头展开后暂时抑制自动收起 |

边缘换尺寸时固定与屏幕接触的那一边，并围绕小窗视觉中心调整另一轴，避免卡片在收起/展开时“跳走”。

### 边缘行为验收

1. 在多显示器和不同缩放比下拖到左、右、上边缘，释放后吸附并自动收起。
2. 进度条态点击展开，窗口完全位于 work area 内；再次失焦或显式收起时回到原边缘。
3. 非边缘位置保持展开，不自行收起。
4. 重启后恢复合法边缘、状态和两态尺寸；旧版自由尺寸归一化。
5. 透明页面无方形底色、无滚动条、无裁切；深浅主题和 running/paused 全部覆盖。

## 7. 可访问性与文案

- 键盘焦点环必须可见；图标按钮提供可访问名称和 tooltip。
- 破坏操作有明确文案，异步操作在执行期间防重复提交但不锁死整个页面。
- 错误提示包含对象、动作和下一步，例如“滴答任务完成失败：CLI 未返回有效结果，可在设置 > 连接查看诊断”。
- 时间、关联和同步三类状态词遵守后端共享契约，禁止用“成功”掩盖只能证明本地写入的结果。

## 8. 明确禁止

- 恢复侧边栏、重复页面标题、独立任务抽屉和任务页两套管理入口。
- 重新引入自由缩放或第三种小窗尺寸。
- 为装饰添加不可操作卡片、假数据图表、英文营销副标题或高频/高亮全屏扫光。
- 在 renderer 中直接读取数据库、文件系统或调用 shell。
- 在不同页面复制任务树、计时累计、同步文案或 CLI 优先/OAuth 后备连接算法。

## 9. 前端验收

- 四个顶级入口均可通过鼠标和键盘访问，切换无布局跳动。
- 亮色与深邃主题覆盖 idle/running/paused、任务列表、任务选择器、统计、设置和小窗。
- 主窗工作面无大面积 blur/光晕，浅色不刺眼、暗色不糊成一块；字号下限、键盘焦点和正文对比均经人工检查。
- 任务真实 UI smoke 覆盖“完成 → 6 秒内撤销 → 再次完成 → 已完成列表找到 → 恢复”；关联并开始与失败回滚均有测试或可重复 smoke 步骤。
- 完成历史默认按 `completedAt` 降序，30/90/365 天范围和 120 项逐步显示均有回归覆盖。
- 统计页快速展开两条会话、请求失败和计时中长时滚动都不会卡死或显示错行详情。
- 统计有记录时必须在同一分析画布内渲染构成、每日趋势与单次排行；切换范围后图表数值、标签和动画从真实筛选结果更新，且三块内容不得表现为相互独立的卡片。
- 字体 smoke 必须验证两种方案的实际 font family 与 tracking 均不相同，不能只检查设置值或按钮选中态。
- 主窗覆盖默认尺寸、最小尺寸和 1280×720；没有横向溢出。
- 小窗覆盖两态、收起信息契约、25/31px 主时间、3px 进度轨、320ms 贴边收束、过渡中拖动取消、四边吸附、多显示器 work area、DPR、实时主题切换和 reduced-motion。
- 完整发布门禁见 [../backend-design/TEST_AND_RELEASE.md](../backend-design/TEST_AND_RELEASE.md)。
