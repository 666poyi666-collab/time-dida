# FocusLink 前端设计规范

> 目标版本：v0.12.0
>
> 当前状态：阶段 1「专注主页与全局动态基础」候选实现，等待视觉验收。
>
> 产品边界：桌面专注计时、任务执行、统计账本与同步控制；不是聊天应用、营销页或通用仪表盘。

## 1. 设计方向

默认视觉主题是「安静的桌面时间仪器」。亮色是设计起点，暗色是同一信息层级和材质逻辑的映射，不得先设计暗色再机械反相。界面应同时具备：

1. Session 式专注仪表、明确的当前任务意图和沉浸构图。
2. Things 3 式连续任务列表、留白、圆形复选框和渐进披露。
3. Rize 式时间趋势、时间构成和单一主分析目标。
4. Raycast 式设置、小窗和高频桌面操作密度。
5. Linear 式弱导航、强内容、统一页头和筛选控件。

表现力可以来自动态光场、玻璃、泛光、渐变边缘、材质高光和连续形变，但不得依赖紫蓝霓虹模板、不可读的多层玻璃、等重卡片网格、chip 墙或没有信息含义的装饰图标。

## 2. 信息架构

主窗口只有四个同级入口：

| 入口 | 主目标 | 明确禁止 |
| --- | --- | --- |
| 专注 | 当前任务、计时、开始/暂停/继续/结束、本次专注账本 | 完整任务管理、重复统计大卡 |
| 任务 | 滴答任务树、搜索、排序、完成/恢复、关联并开始 | 每条任务一张卡、暴露 Provider 选择 |
| 统计 | 时间去向与专注稳定性 | 复杂后台仪表盘、假数据 |
| 设置 | 连接、同步、体验、主题、小窗和版本 | 每个设置项一张卡、首屏原始诊断 |

全局导航保持窄侧轨：品牌、四个入口、当前计时状态。侧轨的视觉重量必须低于页面主内容。

## 3. 主题、颜色与字体

### 3.1 主题模型

- `themeFamily`: `quiet | dawn | bloom`
- `theme`: `light | dark | system`
- 新安装默认 `quiet + light`。
- Dawn Ledger 与 Bloom Console 保留原有气质，但共享主题、动效和状态语义基础。
- 小窗可跟随主窗口或独立选择亮、暗、系统。
- legacy 主题继续兼容 `accentColor` 与 `fontProfile`；Quiet 固定字体与主色。
- legacy 主强调色为红色时，暂停色自动切换为 warning 对比色。

### 3.2 Quiet 色板

| Token | 亮色 | 暗色 |
| --- | --- | --- |
| canvas | `#F3F1EC` | `#111411` |
| surface | `#FAF9F6` | `#171B18` |
| raised | `#FFFFFF` | `#202521` |
| text-primary | `#20221F` | `#F0F1ED` |
| text-secondary | `#747871` | `#969D96` |
| border | `#DCDDD7` | `#2B312D` |
| accent | `#286C63` | `#78C5B5` |
| accent-soft | `#DDEAE6` | `#203C36` |
| pause | `#CC5145` | `#EF6A5C` |
| warning | `#B47A2B` | `#D8AA57` |

全部颜色、透明度、渐变节点、阴影、边缘高光和 Canvas 光场颜色必须从 `foundation.css` 的语义 token 派生。组件不得散落硬编码主题色。

### 3.3 字体

- Quiet 界面：IBM Plex Sans。
- 中文回退：Microsoft YaHei UI、Noto Sans SC。
- 时间和关键数字：IBM Plex Mono。
- 所有时间、时长和统计数字使用 `tabular-nums`。
- 建立页面标题、主数据、正文、元信息和诊断文字五级层级。
- 主窗唯一信息不得小于 11px；普通正文为 13–14px。

## 4. 全局动态与材质系统

### 4.1 Canvas 环境光场

主窗口只运行一个共享 Canvas 动画循环：

- 暖白基础材质缓慢变化。
- 指针光场使用弹簧、阻尼、速度拉伸和反向视差，不直接黏住光标。
- 待机为低频均匀呼吸；运行使用 accent 定向流动；暂停减速并切换 pause 色；完成后收束回待机。
- 专注页强度最高；统计页与数据变化轻度联动；任务和设置降低背景存在感；小窗使用轻量同源材质。
- Dawn 使用暖纸/余烬色温，Bloom 使用冷色曦光，Quiet 使用象牙白和深青绿。
- Canvas 不可用时保留三个 CSS 光斑作为可用降级，不得黑屏。
- Canvas DPR 上限为 2；指针热路径不得读取布局。
- 页面隐藏时停止绘制，重新显示后恢复。

### 4.2 材质规则

- 玻璃只服务导航、账本、弹层等确有前后层关系的区域，不覆盖所有内容。
- 光效经过文字区域时仍需保证可读性；动态背景不得承担信息表达。
- 边缘高光、内反射和外部泛光必须来自 token。
- 不使用高频全屏扫光、发光文字、彩虹边框或粒子遮挡。
- 关键状态不能只靠颜色，必须同时有文案、图形或位置变化。

### 4.3 动效节奏

| Token | 基准 | 场景 |
| --- | --- | --- |
| fast | 100–140ms | 按下、hover、即时反馈 |
| normal | 180–240ms | 控件状态、列表更新 |
| slow | 320–460ms | 页面、弹层、图表形变 |
| ambient | 20s 以上 | 环境材质与低频呼吸 |

- 导航指示器连续移动，页面切换方向与导航顺序一致。
- 按钮 hover 移动局部高光，active 压缩回弹，disabled 停止动态。
- 复选框先完成勾线和填充，再改变任务行位置。
- 图表从基线、起点或圆弧零点连续绘制；范围切换不整块闪烁。
- 页面退出层立即失去 pointer events。
- 关键操作在 100ms 内出现视觉反馈，动效不得延迟业务操作。
- `prefers-reduced-motion` 停止持续位移、流体、呼吸和弹性，仅保留必要状态结果。

## 5. 专注页

专注页使用约 58%/42% 双区：

### 左侧仪器

- 当前任务意图。
- 约 300px、60 根发丝刻度的圆形计时仪表。
- 84px IBM Plex Mono 主计时数字。
- 状态、片段编号和开始/暂停时间。
- 开始、暂停、继续、结束操作。
- 累计专注、累计暂停、总历时共享同一垂直轴。

开始/继续使用 accent，暂停使用 pause 红，结束使用中性 ink。按钮必须具有 hover、active、focus-visible 和 disabled。仪表刻度、针、读数、活动轨和背景光场随计时状态一致变化。

### 右侧账本

- 当前任务。
- 当前 Segment 及完整起止时间和持续时长。
- 历史 Segment。
- 暂停区间。
- 专注与暂停按发生顺序交织。

账本使用文本、发丝线、窄状态边和轻量材质，不使用 chip 墙或相互竞争的卡片。时间列必须保留右侧安全间距，不得贴边或截断。

## 6. 任务页目标

- 使用 Things 3 式连续列表，不为每项创建卡片。
- 使用圆形 accent 复选框；完成动画先给反馈，再收束任务行。
- 标题是第一层级，清单、时间、标签和子任务为第二层级。
- 删除含义不明的棋子或旗帜；优先级使用有文字含义的低权重标记。
- 操作在 hover、键盘焦点或当前选中后渐进显示。
- 保持搜索、排序、任务树、关联并开始、6 秒撤销、30/90/365 天完成范围和 120 项逐步加载。

## 7. 统计页目标

- 默认今天，支持前一天/后一天，今天禁用未来方向。
- 范围为今天、近 7 天、近 15 天、近 30 天和自定义。
- 第一屏只回答“时间花在哪里”和“专注是否稳定”。
- 使用每日柱状趋势、专注/暂停环形构成、任务时间水平分布、当天混合时间轴和单次专注强度。
- 每次强度必须显示完整开始–结束时间。
- 图表支持鼠标和键盘读取精确值；空数据不填假图。
- 快速切换范围时旧请求不得覆盖新结果。

## 8. 设置与小窗目标

设置采用 Raycast 式紧凑分组列表。主题家族和外观模式分开选择；开关统一为 42×24px，关闭态也必须有清楚边界。

小窗尺寸唯一真值是 `shared/miniWindowLayout.ts`：

- 收起 `184×35`：状态、当前时间、3px 进度轨、展开入口。
- 展开 `256×92`：任务、当前时间、累计专注/暂停/总历时和全部控制。
- 不增加第三尺寸，不允许自由缩放。
- 原生拖拽释放后先吸附，再播放 320ms 收束并自动折叠。
- 展开向当前显示器内部生长并限制在 work area。
- 动态材质不得影响拖拽区域、点击命中和数字清晰度。

## 9. 通用组件与状态

以下内容必须属于统一设计系统，而不是页面临时样式：

- Tooltip、Toast、菜单、日期选择器、确认弹窗。
- Loading、Skeleton、空数据、首次使用、错误、断线和重试。
- 滚动条、文本选择、窗口失焦和拖拽。
- Hover、active、focus-visible、disabled、selected、destructive。
- 中英文混排、超长任务、大数字、零数据和极端数据。

关联与同步文案必须遵守共享契约：

- `已关联 / 未关联` 只表示本地任务关联。
- `已同步 / 未同步 / 同步失败` 只表示滴答云同步状态。
- 番茄 To-do 使用“已写入本地 / 待上传 / 上传已确认”，不能宣称独立云端回读。

## 10. 阶段验收

每阶段必须：

1. 提供可运行 Electron 审阅构建，以实际操作手感为主要依据。
2. 提供亮暗状态截图作为排版、溢出和文字清晰度证据。
3. 覆盖 hover、active、focus-visible、disabled、空状态和长文本。
4. 检查默认尺寸、最小尺寸和 1280×720。
5. 运行 format、typecheck、lint、test 和 build。
6. 更新本规范对应章节。
7. 用户明确批准后才进入下一阶段。

阶段 1 截图必须覆盖 idle、running、paused 和 multi-segment；应用体验必须验证指针光场的惯性、状态颜色变化、页面隐藏恢复和 reduced-motion。
