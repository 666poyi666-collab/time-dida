# FocusLink 前端设计规范

> 目标版本：v0.12.4
>
> 当前状态：Linear Workbench 重构已实现；以连续工作面、固定侧轨与流动时间织带为核心。
>
> 产品边界：桌面专注计时、任务执行、统计账本与同步控制；不是聊天应用、营销页或通用仪表盘。

## 1. 设计方向

默认视觉系统是原创的「线性时间台（Linear Workbench）」。所有页面共享一张连续工作面，以发丝线、固定列和留白建立层级，不再用圆角卡片拼接页面。

1. 主窗口使用无边框自绘窗口；导航是 68px 固定侧轨，窄窗时变为底轨。
2. 专注页使用巨大数字排版与真实时间织带；禁止圆钟、速度表和伪百分比仪表。
3. 任务页使用连续列表、克制元信息和渐进操作，不把任务包装成卡片。
4. 统计页是一张连续分析画布，柱、趋势、构成、排行和混合时间轴共享同一网格。
5. 设置页是桌面检查器式分组长页；小窗是同源的两尺寸时间票据。

表现力来自稳定网格、状态色、方向明确的时间动画和精确排版。禁止紫蓝霓虹模板、Bento 卡片墙、玻璃胶囊、chip 墙和没有信息含义的装饰图标。

## 2. 信息架构

主窗口只有四个同级入口：

| 入口 | 主目标 | 明确禁止 |
| --- | --- | --- |
| 专注 | 当前任务、计时、开始/暂停/继续/结束、本次专注账本 | 完整任务管理、重复统计大卡 |
| 任务 | 滴答任务树、搜索、排序、完成/恢复、关联并开始 | 每条任务一张卡、暴露 Provider 选择 |
| 统计 | 时间去向与专注稳定性 | 复杂后台仪表盘、假数据 |
| 设置 | 连接、同步、体验、主题、小窗和版本 | 每个设置项一张卡、首屏原始诊断 |

全局导航使用左侧 68px 固定侧轨，包含品牌、四个入口与计时状态点；900px 以下变为底部固定轨。窗口顶部仅保留透明拖拽热区与右上角三个直角自绘窗口按钮，不形成标题栏或内容导航行。所有按钮明确排除拖拽命中。

## 3. 主题、颜色与字体

### 3.1 主题模型

- `theme`: `light | dark | system`
- 新安装默认 `light`。
- 产品只有一套视觉语言；旧 `themeFamily` 与 `accentColor` 仅作为设置兼容字段保留，运行时统一解析，不在设置中暴露。
- 小窗可跟随主窗口或独立选择亮、暗、系统。
- `fontProfile` 可调整排版气质，不改变布局和固定状态色。

### 3.2 单一语义色板

| Token | 亮色 | 暗色 |
| --- | --- | --- |
| canvas | `#F4F3EF` | `#0F1114` |
| surface | `#FAF9F6` | `#15181C` |
| raised | `#FFFFFD` | `#1D2025` |
| text-primary | `#181A1D` | `#EFEFEB` |
| text-secondary | `#646667` | `#9E9F9C` |
| border | `#D6D4CD` | `#2F3236` |
| interface | `#2563EB` | `#60A5FA` |
| interface-soft | `#DBEAFE` | `#1E3A69` |
| focus | `#059669` | `#34D399` |
| pause | `#D97706` | `#FBBF24` |
| warning | `#B26F1D` | `#E1AA5A` |

全部颜色、透明度、渐变节点、阴影、边缘高光和 Canvas 光场颜色必须从 `temporal-foundation.css` 的语义 token 派生。组件不得散落硬编码主题色。

### 3.3 字体

- `plex / 精密`：IBM Plex Sans + IBM Plex Mono，默认配置，偏仪器感与中文秩序。
- `manrope / 人文`：Manrope + Space Grotesk，较舒展、适合长时间阅读。
- `geist / 现代`：Geist + JetBrains Mono，紧凑、信息密度高。
- `sora / 几何`：Sora + JetBrains Mono，轮廓鲜明，关键数字更醒目。
- 中文统一回退：Microsoft YaHei UI、Noto Sans SC。
- 所有时间、时长和统计数字使用 `tabular-nums`。
- 建立页面标题、主数据、正文、元信息和诊断文字五级层级。
- 主窗唯一信息不得小于 11px；普通正文为 13–14px。

## 4. 全局动态与材质系统

### 4.1 Canvas 环境光场

主窗口只运行一个共享 Canvas 动画循环：

- 暖白基础材质缓慢变化。
- 指针光场使用弹簧、阻尼、速度拉伸和反向视差，不直接黏住光标。
- 待机为低频均匀呼吸；运行使用 accent 定向流动；暂停减速并切换 pause 色；完成后收束回待机。
- 专注页强度最高；统计页与数据变化轻度联动；任务和设置降低背景存在感；小窗使用轻量同源材质。
- 环境层只使用当前明暗外观的中性画布与低透明界面蓝，不改变状态色。
- Canvas 不可用时保留三个 CSS 光斑作为可用降级，不得黑屏。
- Canvas DPR 上限为 2；指针热路径不得读取布局。
- 页面隐藏时停止绘制，重新显示后恢复。

### 4.2 材质规则

- 主导航、账本和页面均使用高不透明平面；只有弹层可使用轻微透明度建立前后关系。
- 光效经过文字区域时仍需保证可读性；动态背景不得承担信息表达。
- 边缘高光、内反射和外部泛光必须来自 token。
- 不使用高频全屏扫光、发光文字、彩虹边框或粒子遮挡。
- 关键状态不能只靠颜色，必须同时有文案、图形或位置变化。

### 4.3 动效节奏

| Token | 基准 | 场景 |
| --- | --- | --- |
| fast | 100–140ms | 按下、hover、即时反馈 |
| normal | 180–240ms | 控件状态、列表更新 |
| slow | 320–460ms | 页面、弹层、图表形变 |
| ambient | 20s 以上 | 环境材质与低频呼吸 |

- 导航选中面连续变形，页面切换方向与导航顺序一致。
- 按钮 hover 移动局部高光，active 压缩回弹，disabled 停止动态。
- 复选框先完成勾线和填充，再改变任务行位置。
- 图表从基线、起点或圆弧零点连续绘制；范围切换不整块闪烁。
- 页面退出层立即失去 pointer events。
- 关键操作在 100ms 内出现视觉反馈，动效不得延迟业务操作。
- `prefers-reduced-motion` 停止持续位移、流体、呼吸和弹性，仅保留必要状态结果。

## 5. 专注页

专注页在有账本时使用约 64%/36% 双区；待机无账本时使用完整宽度：

### 左侧时间稿件

- 当前任务标题是页面第一语言层级，默认文案为“给这一段时间一个名字”。
- 70–150px 等宽主计时数字构成视觉锚点；不使用钟面。
- 时间织带把当前时刻固定在 76% 位置；世界时间、刻度与真实事件每秒向左推进。
- 专注使用连续绿色带，暂停使用琥珀斜纹；暂停时专注段冻结在暂停边界，暂停段继续向“此刻”生长，恢复后由新专注段接力。
- 时间织带同时显示固定 NOW 游标、动态刻度、真实事件段和底部总历时构成轨；构成轨只表达已经发生的专注/暂停比例。
- 状态、片段编号和开始/暂停时间。
- 开始、暂停、继续、结束操作。
- 累计专注、累计暂停、总历时共享同一垂直轴。

开始/继续使用界面蓝，暂停操作使用中性 ink，结束使用中性描边；状态读数和时间数据使用可配置专注绿或暂停红。按钮必须具有 hover、active、focus-visible 和 disabled。不得重新引入圆形表盘，也不得伪造倒计时目标或任务完成百分比。

### 右侧账本

- 当前任务。
- 当前 Segment 及完整起止时间和持续时长。
- 历史 Segment。
- 暂停区间。
- 专注与暂停按发生顺序交织。

账本使用文本、发丝线、窄状态边和轻量材质，不使用 chip 墙或相互竞争的卡片。时间列必须保留右侧安全间距，不得贴边或截断。

## 6. 任务页目标

- 使用 Things 3 式连续列表，不为每项创建卡片。
- 使用圆形 accent 复选框；完成动画先给反馈，再收束任务行。
- 标题是第一层级，清单、时间、标签和子任务为第二层级。
- 删除含义不明的棋子或旗帜；优先级使用有文字含义的低权重标记。
- 操作在 hover、键盘焦点或当前选中后渐进显示。
- 保持搜索、排序、任务树、关联并开始、6 秒撤销、30/90/365 天完成范围和 120 项逐步加载。

## 7. 统计页目标

- 默认今天，支持前一天/后一天，今天禁用未来方向。
- 范围为今天、近 7 天、近 15 天、近 30 天和自定义。
- 第一屏只回答“时间花在哪里”和“专注是否稳定”。
- 单日使用带 00–24 时坐标的 24 小时专注/暂停节律柱；多日使用带基线与常驻读法的每日柱状趋势。
- 使用直角双轨构成环、任务时间水平分布、带 0/6/12/18/24 标签的当天混合时间轴和单次专注强度；零占比轨道不得残留圆点。
- 每次强度必须显示完整开始–结束时间。
- 图表支持鼠标和键盘读取精确值；空数据不填假图。
- 快速切换范围时旧请求不得覆盖新结果。

## 8. 设置与小窗目标

设置采用紧凑分组列表。只提供明亮、深色、跟随系统三种外观；开关统一为 42×24px，关闭态也必须有清楚边界。

- 设置默认进入“界面与体验”，再进入连接或同步。
- Plex/Geist/Manrope/Sora 必须以真实字体、字重和字距渲染大字号预览，不只改变选项名称。

小窗尺寸唯一真值是 `shared/miniWindowLayout.ts`：

- 收起 `184×35`：状态、当前时间、3px 进度轨、展开入口。
- 展开 `320×124`：任务可完整换行显示，并同时容纳当前时间、累计专注/暂停/总历时和全部控制。
- 不增加第三尺寸，不允许自由缩放。
- 原生拖拽释放后先吸附，再播放 320ms 收束并自动折叠。
- 展开向当前显示器内部生长并限制在 work area。
- 小窗使用平面实色表面、1px 边框和顶部状态边，不使用玻璃、光晕或胶囊外形。
- 运行态当前时间使用 focus 绿，暂停态当前时间使用 pause 琥珀；“暂停”按钮保持中性 ink，“继续”按钮使用界面蓝，避免把暂停操作误表现为破坏性动作。

## 9. 通用组件与状态

以下内容必须属于统一设计系统，而不是页面临时样式：

- Tooltip、Toast、菜单、日期选择器、确认弹窗。
- Loading、Skeleton、空数据、首次使用、错误、断线和重试。
- 滚动条、文本选择、窗口失焦和拖拽。
- Hover、active、focus-visible、disabled、selected、destructive。
- 中英文混排、超长任务、大数字、零数据和极端数据。

关联与同步文案必须遵守共享契约：

- `已关联 / 未关联` 只表示本地任务关联。
- `已同步 / 未同步 / 同步失败` 只表示滴答云同步状态。
- 番茄 To-do 使用“已写入本地 / 待上传 / 上传已确认”，不能宣称独立云端回读。

## 10. 阶段验收

每阶段必须：

1. 提供可运行 Electron 审阅构建，以实际操作手感为主要依据。
2. 提供亮暗状态截图作为排版、溢出和文字清晰度证据。
3. 覆盖 hover、active、focus-visible、disabled、空状态和长文本。
4. 检查默认尺寸、最小尺寸和 1280×720。
5. 运行 format、typecheck、lint、test 和 build。
6. 更新本规范对应章节。
7. 用户明确批准后才进入下一阶段。

阶段 1 截图必须覆盖 idle、running、paused 和 multi-segment；应用体验必须验证指针光场的惯性、状态颜色变化、页面隐藏恢复和 reduced-motion。
