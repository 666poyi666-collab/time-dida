# FocusLink 前端设计规范

> 目标版本：v0.12.x（当前实现：v0.12.17）
>
> 当前状态：「时间仪器 Time Instrument」设计系统已实现并取代 Linear Workbench。
>
> 产品边界：桌面专注计时、任务执行、统计账本与同步控制；不是聊天应用、营销页或通用仪表盘。

## 1. 设计方向

默认视觉系统是「时间仪器」：一张连续、低噪声的浅色工作台，上面只放一台正在运转的精密时间仪器。

1. 主窗口 = 64px 导航轨 + 连续工作面，区域之间只有 1px 发丝线，不用圆角卡片拼接页面。面板圆角恒为 0；按钮、输入、弹层最大 3–6px。
2. 时间之带是全局唯一持续运动的元素；其余动效 ≤360ms，只出现在状态转换。
3. 颜色即语义：用户选择的强调色 = 导航、操作、选中、统计与专注，暂停红 = 损耗，深红 = 危险操作。空闲与未来时间使用中性灰。
4. 统计页是一张连续分析画布，所有图表共享时间尺度、颜色语义与排版规则，并逐块标注计算口径。
5. 特色只通过计时仪表样式、专注强调色与少量显示偏好体现，不存在多套主题。

禁止：圆角卡片墙、玻璃拟态、发光/霓虹、大面积渐变、装饰性环形图、假数据与平滑伪造曲线、把暂停或空闲状态画成蓝色。

## 2. 信息架构

主窗口只有四个同级入口：

| 入口 | 主目标 | 明确禁止 |
| --- | --- | --- |
| 专注 | 当前任务、计时仪表、开始/暂停/继续/结束、本次专注账本 | 完整任务管理、重复统计大卡 |
| 任务 | 滴答任务树、搜索、排序、完成/恢复、关联并开始 | 每条任务一张卡、暴露 Provider 选择 |
| 统计 | 时间去向、专注节律与单次质量 | 企业后台模板、假数据 |
| 设置 | 连接、同步、体验、专注仪表、小窗和版本 | 每个设置项一张卡、首屏原始诊断 |

导航轨 64px：顶部品牌标，4 个 56px 导航项（18px 图标 + 常驻 10px 中文标签，不用 tooltip），底部计时状态点。选中态 = 当前强调色文字/图标 + `--app-accent-soft` 底 + 底部居中 2px 强调色短刻度（20px 宽）；不使用左侧竖条或固定蓝色。900px 以下变为底部横轨。

页面切换、普通操作、危险操作三类按钮严格分离：主操作 = 当前强调色实心，暂停操作 = 中性墨色，次操作 = 描边，危险 = 深红实心 + 二次确认。

## 3. 主题、颜色与字体

### 3.1 主题模型

- `theme`: `light | dark | system`，新安装默认 `light`。
- 亮色是主要设计；暗色是同一 token 体系的映射，不是另一套皮肤。
- 旧 `themeFamily` / `accentColor` 仅作为兼容字段保留；旧字体值迁移到 Noto Sans SC。旧 `timerStyle`（editorial/digital/mono）在启动时一次性迁移（editorial→thin，digital→pixel，mono→standard），迁移失败回落 `standard`，不得启动白屏。

| surface | `#FCFCFA` | `#15181C` |
| surface-2 | `#F0F0EC` | `#1D2025` |
| text-primary | `#181A1D` | `#EFEFEB` |
| text-secondary | `#5F6368` | `#9E9F9C` |

| border | `#DDDCD6` | `#2F3236` |
| interface | `#2563EB` | `#60A5FA` |
| interface-soft | `#E7EEFC` | `#1E3A69` |
| focus（默认翡翠） | `#0E9F6E` | `#34D399` |
| pause | `#D24339` | `#F47067` |
| danger | `#A02B20` | `#FF8A80` |

- 全部颜色从 `src/styles/temporal-foundation.css` 的语义 token 派生；组件不得散落硬编码主题色。
- 强调色跨色相五选（翡翠/钴蓝/鸢尾/琥珀/石墨），同时映射 `--app-accent*` 与 `--app-success*`：导航、按钮、链接、任务选中态、统计图、运行读数、时间之带和账本标题必须使用同一色相；暂停始终为红色。
- 暂停红用于文字、轨迹、斜纹；危险深红永远实心填充 + 二次确认，两者永不相邻使用。
- `temporal-foundation.css`（token）+ `legacy-support.css`（任务/设置/浮层沿用规则的清理版）+ `linear-workbench.css`（单一表现层）是当前全部样式来源；`temporal-ui.css` 与 AmbientField 已删除。

界面字体与仪表字体使用真实不同的字形骨架，全部本地嵌入：

| 中文正文方案 C | LXGW Neo ZhiSong（霞鹜新致宋） | OFL |
| 中文正文方案 D | LXGW Marker Gothic（霞鹜漫黑） | OFL |
| 中文正文方案 E | LXGW Neo XiHei（霞鹜新晰黑，正线体骨架） | OFL |
| 中文正文方案 F | Smiley Sans（得意黑，倾斜几何骨架） | OFL |
| 数据/账本/常规计时数字 | JetBrains Mono 400/500/600 | OFL |

- 正文与普通标签用正常字重；标题适度强调；极细样式必须保持纤细；点阵靠网格构成，翻页靠结构与运动，均不靠加粗。
- 主窗唯一信息字号不小于 11px；10px 仅用于非核心辅助标签。

### 3.4 动效节奏

| 场景 | 参数 |
| --- | --- |
| 时间之带逐秒步进 | 130ms，cubic-bezier(.16,.9,.3,1)，与真实秒边界对齐 |
| 时间之带近远变焦 | 820ms，对数空间插值，easeInOutQuart；中段保持空间连续，不做换图 |
| hover/按下 | 120ms ease-out |
| 控件/弹层 | 200ms ease-out |
| 页面/账本滑入 | 360ms |
| 翻页机械翻牌 | fold 170ms + unfold 190ms，由 `animationend` 驱动提交 |
| 像素点亮 | 140ms opacity 过渡 |

`prefers-reduced-motion`：时间之带取消步进与变焦（瞬时切换），翻牌改为静态数字提交，像素点亮与环境动效关闭；运行中切换系统偏好必须立即生效，信息表达不损失。

## 4. 专注页

信息层级固定为：当前状态 → 当前任务 → 主计时仪表 → 时间之带 → 累计三项 → 控制 → 右侧账本。

- 主计时读数：running = 当前片段时长（专注色），paused = 当前暂停经过（暂停红，继续滴答），idle = 墨色 `00:00`。暂停时读数不降透明度；标签行显示「暂停于 HH:MM · 有效专注 X 已冻结」。
- 右侧账本与主区同一平面（360px，1600px 以上 420px），1px 发丝竖线分隔，不是浮层。条目 = 时间脊线锚点（HH:MM）+ 序号 + 类型（专注绿/暂停红）+ 时长 + 任务 + 关联/同步语义；进行中条目用浅底强调，时长逐秒刷新。
- 累计专注（绿）/累计暂停（红）/总历时（墨）横排，竖发丝线分隔。
- 账本标题与摘要：running = 当前强调色，paused = 暂停红；正文与分隔仍保持中性，避免整栏染色。
- 控制：暂停 = 墨黑实心，继续 = 当前强调色实心，结束 = 常驻描边。禁止把继续画成暂停红。
- 大窗口（≥1600px）：读数、时间之带、账本全部加档，不允许「细长内容放空」。1280×720 低保档压缩纵向空间但保留全部元素。

### 4.1 五套计时仪表（`timerStyle` 可保存设置）

| 样式 | 构成 |
| --- | --- |
| standard 标准等宽 | JetBrains Mono 固定数字槽 + `ELAPSED / LIVE / HOLD / READY` 工业读数标记，默认 |
| flip 翻页机械 | Oswald + 上下分片卡片 + 中央转轴线；`steady → fold → unfold → commit` 由动画结束事件推进，动画中只保留最新目标；idle/finished 静态提交 |
| pixel 像素点阵 | 自绘 7×9 整数网格数字（`crispEdges`，熄灭格 2.5% 墨），右侧「专注核心」随累计专注点亮（45 分钟充能一整颗，比例可重入） |
| thin 高反差编辑 | Bodoni Moda，衬线粗细对比与编辑排版气质 |
| segment 七段数码 | 自绘 SVG 七段结构，使用真实段码开关，不依赖近似数码字体 |

四套特色仪表与标准读数共享：当前任务、当前状态、累计专注/暂停/总历时、控制与时间之带。

### 4.2 沉浸模式

专注页「全屏沉浸」按钮同时切换 Electron 原生全屏与 body 覆盖层；Esc 键或右上角「Esc 退出」按钮离开，进入与退出不改变计时状态。操作系统拒绝原生全屏时，覆盖层仍提供完整的窗口内沉浸。

- 自上而下固定为：状态与当前任务 → 主计时仪表（五套样式均可用，读数随视口放大）→ 累计专注/暂停/总历时与全部控制 → 底部放大的时间之带。
- 时间之带约占 1/3 屏高（36vh，最小 240px），与专注页共享同一 canvas 契约：逐秒步进、暂停变焦与 reduced-motion 行为完全一致。
- 沉浸中不显示导航轨与右侧账本；覆盖层复用同一 TimerDial 与 TemporalRibbon 实现，不存在第二套仪表代码。
- 普通窗口中的右侧专注账本可折叠，释放水平空间；折叠不改变会话和片段数据。

## 5. 时间之带

刻度本身就是进度：颜色进度充满整个刻度区域，刻度绘制在颜色区域之上，没有第二根独立进度条。

- 区域语义：已专注 = 专注色实体（顶部 2px 深边）；暂停 = 红色斜纹 + 浅红底；未发生 = 中性灰；未来（指针右侧）= 压暗灰。刻度线使用墨色透明度，在任何区域上可辨认。
- 指针固定在带宽 62% 处，2px 墨色 + 「现在」标签；运行/暂停时指针根部的状态色圆点每秒一次脉冲。
- 逐秒擒纵：秒边界后 130ms 内世界向左步进一格，其余时间静止；不是连续漂移。
- 专注 = 秒级近景（8px/秒：小格 1 秒、中格 5 秒、大格 1 分钟带时刻标签）；暂停/空闲/结束 = 远景（30 分钟大格 ≈420px，5 分钟小格）。
- 顶部提供「近景 / 远景 / 跟随状态」控制；跟随状态时专注自动近景、暂停自动远景，手动选择后保持到用户恢复跟随。
- 专注↔暂停切换 = 同一世界坐标系下 720ms 对数变焦（scale 连续插值、秒刻度随密度淡出、分钟刻度淡入），历史片段不销毁、不重排；专注与暂停在任意尺度都保持秒级更新。
- 当前边界绘制状态粒子：运行仅保留克制尾尘；暂停从边界产生红色方形碎片，向已消耗方向侵蚀、脱落和消失，并显示警惕文案。粒子不能遮挡刻度与标签。
- 渲染为单一 canvas（`shared/focus/bandMath.ts` 为纯函数内核），暂停期间指针继续随真实时间前进，暂停段从暂停起点向指针生长。
- idle/finished 以最后一条记录为锚点完全冻结，不再跟随系统时钟漂移；非活动且变焦完成后停止 60fps 重绘，只在尺寸、主题或数据变化时补一帧。
- reduced-motion：瞬时切换尺度，保留每秒数据更新。

## 6. 任务页目标

保持现有任务布局与信息结构：左侧任务索引、右侧执行序列/完成档案。

- 连续密排行（无卡片），行高与层级靠缩进与发丝线；当前任务行 = 浅蓝底 + 文字标签，不用左缘竖线。
- 优先级用文字标记（高/中/低），逾期用 warning 色。
- 「开始专注」在行 hover/键盘焦点时渐进显示，不每行常驻大按钮。
- 保持搜索、排序、任务树、关联并开始、6 秒撤销、30/90/365 天完成范围和 120 项逐步加载。

## 7. 统计页目标

统计采用熟悉的顺读日报，不提供抽象可视化切换器：

1. **结论区**：大号有效专注读数 + 一句自然语言结论 + 专注率仪表，明确日期和计算口径。
2. **四项 KPI**：有效专注、暂停损耗、完成轮次/活跃天数、最长一轮；共享一条连续分区，不拆成浮动卡片。
3. **单日时间轴**：上方固定 00/06/12/18/24 的全天定位轨，下方自动放大活跃时段的专注/暂停双车道；旧记录没有片段时显示明确说明，不画空轨。
4. **多日趋势**：使用带 0/中值/最大时长刻度的每日堆叠柱，强调色为专注、红色为暂停；每一天可悬停或键盘聚焦读取精确值，不伪造平滑曲线。
5. **任务投入构成**：100% 构成带 + 精确时长/占比行，最大余数法保证显示百分比合计 100%；已关联、未关联、旧会话汇总分别标记。
6. **暂停损耗**：页尾用专注/暂停守恒轨与每轮平均专注收束结论；最近会话只保留下方唯一账本，Dashboard 内不重复表格。

范围：单日（可前后翻页，不进未来）、近 7 天、半个月、1 个月、自定义。旧请求不得覆盖新结果（request-id 保护保持）。

## 8. 设置与小窗目标

设置页「界面与体验」包含：外观（明亮/深色/跟随系统）、界面字体（六套真实不同骨架）、全局强调色（五选）、计时仪表（五卡实时渲染预览，非截图）、状态色说明、小窗与沉浸偏好。

设置页「同步」提供默认关闭的「PC 参与实时专注」。开启后必须明确云端是活动计时唯一事实源、断线时控制不可确认，并显示实时连接、revision 与状态；不得继续展示“运行中的计时只归当前设备”。切换事实源只能在本地计时 idle 时生效。

小窗尺寸唯一真值是 `shared/miniWindowLayout.ts`：

- 收起 `184×35`：状态点 + 状态词、当前时间（25px）、2px 进度轨与状态粒子、展开入口。
- 展开 `256×70`：上栏容纳状态、完整任务与窗口操作；下方以主时间/60 格秒轨和三项累计/全套控制组成紧密双区。结束态主读数显示本轮累计专注而非最后一段。
- 不增加第三尺寸，不允许自由缩放；旧设置中的 320×124 由 `snapMiniWindowSize` 自动归一。
- 原生拖拽释放后先吸附再 320ms 收束自动折叠；展开向显示器内部生长并 clamp 到 work area；平面实色 + 1px 边框 + 顶部状态边，禁玻璃/光晕/胶囊。
- 时间与按钮分属独立网格行，结构上不允许重叠或换行；暂停粒子必须从当前分钟进度边界消蚀，不得固定从轨道最右端喷出。长任务名自动往返，reduced-motion 下提供可聚焦横向滚动；字体切换后必须重新测量。

应用图标与托盘图标共用 `F / L` 双织带字标：F 带秒刻痕，L 穿过 F 并承担当前状态色。托盘前景随系统明暗切换，运行使用专注色、暂停使用暂停红；禁止退回通用圆环、番茄或脉搏曲线图标。

## 9. Web / Android 多端专注控制台

Web/PWA 与 Capacitor Android 继续复用同一套 `src/mobile/` React 界面，但产品能力从只读账本升级为
「实时专注控制台 + 已结束会话账本」。它不复制 Electron renderer；PC 显式启用实时控制后，三端共同控制同一场活动会话：

- 首屏优先展示唯一活动会话。状态固定为待机、专注中、已暂停；可填写本次标题并开始，也可暂停、继续和结束。任何写操作等待云端确认期间都必须防重复提交。
- 实时状态由云端 revision 作为权威事实。客户端以响应 `serverTime` 为显示基点逐秒外推三时间；断线时明确标记「离线推算」，不得把本机 tick 冒充另一设备已经确认的状态。
- 所有命令携带幂等 command id 与 `expectedRevision`。另一设备已经抢先操作时，本机刷新最新快照并显示「状态已由另一设备更新」，不静默覆盖。
- 在线时通过有界长轮询等待 revision 变化；页面隐藏、网络切换、连接参数变化和组件卸载必须取消旧请求，重连不得累加轮询或让旧账号数据回填。
- 结束会话后继续复用完整的 Session / Segment / Pause 账本与 IndexedDB 缓存。关联状态仍使用「已关联 / 未关联」，来源设备不能写成任务来源；缓存命中、实时状态确认和账本拉取是三种不同结果。
- 移动端不直接调用 dida CLI 或番茄 To-do 本地桥。结束记录会进入 FocusLink 跨设备账本；只有桌面端先导入权威 completed bundle，再真实执行并确认第三方投递后，才能显示对应的「已同步」或「上传已确认」。PC 未开启实时控制时仍保持本地优先，不得暗示移动端已接管 PC 计时。
- 手机使用单列顺读布局；平板从 760px 起使用控制台与账本双列，横屏允许控制区保持可见。所有输入和动作至少 44px，文本输入不小于 16px；使用 `100dvh`、safe-area 与 320px 下限，360×800、412×915 及平板横竖屏不得横向溢出。
- 访问令牌默认只存 `sessionStorage`；只有用户勾选「在此设备记住令牌」才写入 `localStorage`，并提供移除令牌与仅清除本机缓存的操作。切换 endpoint 或 token 必须先清空旧账号缓存与实时快照。
- PWA manifest、页面标题与离线 app shell 使用「FocusLink 专注」语义，不再写「只读预览」。Service Worker 只缓存同源静态资源，不缓存 Bearer 响应或跨源实时接口。
- Capacitor 仅包装同一个 `dist-mobile`，Android 工程只提供薄原生集成。最低 API 24；禁止系统备份 WebView token/账本缓存，Network Security Config 仅为 localhost/127.0.0.1 回环测试开放明文，生产地址必须 HTTPS。
- Android 活动会话使用可见前台通知，动作包含暂停/继续/结束；快捷设置 Tile 在已有活动快照时只转发暂停/继续，待机时打开 App 完成必要输入。原生层不推进业务时钟，也不先行改变状态；通知/Tile 动作进入至少一次命令队列，携带 session/revision，并在云端确认或明确冲突处理后由 Web 层确认移除。
- `cloud/` 当前仍是回环优先测试后端，不具备生产身份、备份、限流与监控；测试通过不等于已上线公共云服务。

## 10. 通用组件与状态

- Tooltip、Toast、菜单、日期选择器、确认弹窗属于统一设计系统（发丝边框、3px 圆角、无玻璃）。
- Loading、Skeleton、空数据、首次使用、错误、断线和重试状态明确。
- 滚动条、文本选择（主工作区时间/任务文本可选中复制）、窗口失焦和拖拽行为一致。
- Hover、active、focus-visible、disabled、selected、destructive 各状态完备。
- 关联与同步文案契约：`已关联/未关联` 只表示本地任务关联；`已同步/未同步/同步失败` 只表示滴答云同步队列；番茄 To-do 使用「已写入本地/待上传/上传已确认」。

## 11. 阶段验收

每阶段必须：

1. 提供可运行 Electron 审阅构建，以实际操作手感为主要依据。
2. 提供亮暗状态截图作为排版、溢出和文字清晰度证据。
3. 覆盖 hover、active、focus-visible、disabled、空状态和长文本。
4. 检查默认尺寸、最小尺寸和 1280×720。
5. 运行 format、typecheck、lint、test 和 build。
6. 更新本规范对应章节。
7. 用户明确批准后才进入下一阶段。

阶段 1 截图必须覆盖 idle、running、paused 和 multi-segment；必须验证时间之带逐秒步进、变焦连续性、页面隐藏恢复与 reduced-motion。
