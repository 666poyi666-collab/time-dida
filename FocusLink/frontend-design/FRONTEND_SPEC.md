# FocusLink 前端设计规范

> 目标版本：v0.12.x（当前实现：v0.12.27）
>
> 当前状态：「时间仪器 Time Instrument」设计系统已实现并取代 Linear Workbench。
>
> 产品边界：桌面专注计时、任务执行、统计账本与同步控制；不是聊天应用、营销页或通用仪表盘。

### 当前三端迭代需求（2026-07-22）

- 桌面、手机与平板继续使用同一产品口径共同迭代；移动端不得复制 Electron 服务或宣称桌面专属能力。
- 统计界面吸收参考项目的高密度时间统计、趋势、热力、时段分布与宽屏时间轴组织方式，但必须使用 FocusLink 的真实账本、自然日裁切、任务关联和同步语义，不照搬品牌皮肤或假数据。
- 桌面统计重点改善顺读层级与宽屏布局；手机和平板统计保持同一分析模型，并分别适配单列触控与宽屏双栏。
- 时间之带与五套计时仪表继续优化逐秒更新的平滑度；不得用高频 React 全树重渲染换取视觉连续，也不得牺牲真实秒边界、冻结态或 reduced-motion。
- 本轮先生成 Windows 安装版供用户覆盖安装并验收；用户确认安装后再继续三端联调。手机和平板交付采用覆盖安装，不清除现有应用数据。

## 1. 设计方向

默认视觉系统是「时间仪器」：一张连续、低噪声的浅色工作台，上面只放一台正在运转的精密时间仪器。

1. 主窗口 = 64px 导航轨 + 连续工作面，区域之间只有 1px 发丝线，不用圆角卡片拼接页面。面板圆角恒为 0；按钮、输入、弹层最大 3–6px。
2. 时间之带是全局唯一持续运动的元素；其余动效 ≤360ms，只出现在状态转换。
3. 颜色即语义：用户选择的强调色 = 导航、操作、选中、统计与专注，暂停红 = 损耗，深红 = 危险操作。空闲与未来时间使用中性灰。
4. 统计页是一张连续分析画布，所有图表共享时间尺度、颜色语义与排版规则，并逐块标注计算口径。
5. 特色只通过计时仪表样式、专注强调色与少量显示偏好体现，不存在多套主题。

禁止：圆角卡片墙、玻璃拟态、发光/霓虹、大面积渐变、装饰性环形图、假数据与平滑伪造曲线、把暂停或空闲状态画成蓝色。

## 2. 信息架构

主窗口只有四个同级入口：

| 入口 | 主目标 | 明确禁止 |
| --- | --- | --- |
| 专注 | 当前任务、计时仪表、开始/暂停/继续/结束、本次专注账本 | 完整任务管理、重复统计大卡 |
| 任务 | 滴答任务树、搜索、排序、完成/恢复、关联并开始 | 每条任务一张卡、暴露 Provider 选择 |
| 统计 | 时间去向、专注节律与单次质量 | 企业后台模板、假数据 |
| 设置 | 连接、同步、体验、专注仪表、小窗和版本 | 每个设置项一张卡、首屏原始诊断 |

导航轨 64px：顶部品牌标，4 个 56px 导航项（18px 图标 + 常驻 10px 中文标签，不用 tooltip），底部计时状态点。选中态 = 当前强调色文字/图标 + `--app-accent-soft` 底 + 底部居中 2px 强调色短刻度（20px 宽）；不使用左侧竖条或固定蓝色。900px 以下变为底部横轨。

页面切换、普通操作、危险操作三类按钮严格分离：主操作 = 当前强调色实心，暂停操作 = 中性墨色，次操作 = 描边，危险 = 深红实心 + 二次确认。

## 3. 主题、颜色与字体

### 3.1 主题模型

- `theme`: `light | dark | system`，新安装默认 `light`。
- 亮色是主要设计；暗色是同一 token 体系的映射，不是另一套皮肤。
- 旧 `themeFamily` / `accentColor` 仅作为兼容字段保留；旧字体值迁移到 Noto Sans SC。旧 `timerStyle`（editorial/digital/mono）在启动时一次性迁移（editorial→thin，digital→pixel，mono→standard），迁移失败回落 `standard`，不得启动白屏。

### 3.2 单一语义色板

| --- | --- | --- |
| canvas | `#F5F5F2` | `#0F1114` |
| surface | `#FCFCFA` | `#15181C` |
| surface-2 | `#F0F0EC` | `#1D2025` |
| text-primary | `#181A1D` | `#EFEFEB` |
| text-secondary | `#5F6368` | `#9E9F9C` |
| text-tertiary | `#9A9DA2` | `#6E7378` |
| border | `#DDDCD6` | `#2F3236` |
| interface | `#2563EB` | `#60A5FA` |
| interface-soft | `#E7EEFC` | `#1E3A69` |
| focus（默认翡翠） | `#0E9F6E` | `#34D399` |
| pause | `#D24339` | `#F47067` |
| danger | `#A02B20` | `#FF8A80` |

| warning | `#B26F1D` | `#E1AA5A` |

- 全部颜色从 `src/styles/temporal-foundation.css` 的语义 token 派生；组件不得散落硬编码主题色。
- 强调色跨色相五选（翡翠/钴蓝/鸢尾/琥珀/石墨），同时映射 `--app-accent*` 与 `--app-success*`：导航、按钮、链接、任务选中态、统计图、运行读数、时间之带和账本标题必须使用同一色相；暂停始终为红色。
- 暂停红用于文字、时间之带前沿的短寿命损耗粒子、竖向暂停指针与 mini 小窗的余烬轨迹；它不得在主时间带形成持久区间、横线或历史痕迹。危险深红永远实心填充 + 二次确认，两者永不相邻使用。
- `temporal-foundation.css`（token）+ `legacy-support.css`（任务/设置/浮层沿用规则的清理版）+ `linear-workbench.css`（单一表现层）是当前全部样式来源；`temporal-ui.css` 与 AmbientField 已删除。

### 3.3 字体系统

界面字体与仪表字体使用真实不同的字形骨架，全部本地嵌入：

| 角色 | 字体 | 许可 |
| --- | --- | --- |
| 中文正文方案 A | Noto Sans SC Variable（思源黑体） | OFL |
| 中文正文方案 B | LXGW WenKai 500（霞鹜文楷） | OFL |
| 中文正文方案 C | LXGW Neo ZhiSong（霞鹜新致宋） | OFL |
| 中文正文方案 D | LXGW Marker Gothic（霞鹜漫黑） | OFL |
| 中文正文方案 E | LXGW Neo XiHei（霞鹜新晰黑，正线体骨架） | OFL |
| 中文正文方案 F | Smiley Sans（得意黑，倾斜几何骨架） | OFL |
| 数据/账本/常规计时数字 | JetBrains Mono 400/500/600 | OFL |
| 高反差编辑计时数字 | Bodoni Moda Variable | OFL |
| 翻页机械计时数字 | Oswald Variable | OFL |
| 像素点阵数字 | 自绘 7×9 整数网格点阵（无字体文件） | 项目自有 |
| 七段数码数字 | 自绘 SVG 段码（无字体文件） | 项目自有 |

- 所有时间与统计数字使用 `tabular-nums`；计时数字宽度任何样式下都稳定，刷新不跳位。
- 所有界面文字的 `letter-spacing` 不得为负值；需要容纳长文本或长时间读数时调整容器约束与动态字号，不以压缩字距换空间。
- 正文与普通标签用正常字重；标题适度强调；极细样式必须保持纤细；点阵靠网格构成，翻页靠结构与运动，均不靠加粗。
- 主窗唯一信息字号不小于 11px；10px 仅用于非核心辅助标签。

### 3.4 动效节奏

| 场景 | 参数 |
| --- | --- |
| 时间之带逐秒步进 | 130ms，cubic-bezier(.16,.9,.3,1)，与真实秒边界对齐 |
| 时间之带近远变焦 | 820ms，对数空间插值，easeInOutQuart；中段保持空间连续，不做换图 |
| hover/按下 | 120ms ease-out |
| 控件/弹层 | 200ms ease-out |
| 页面/账本滑入 | 360ms |
| 翻页机械翻牌 | fold 170ms + unfold 190ms，由 `animationend` 驱动提交 |
| 像素点亮 | 140ms opacity 过渡 |

`prefers-reduced-motion`：时间之带取消步进与变焦（瞬时切换），翻牌改为静态数字提交，像素点亮与环境动效关闭；运行中切换系统偏好必须立即生效，信息表达不损失。

## 4. 动效与光效语言

所有过渡共用一套语言：丝滑、短促、只在状态转换处出现。时间之带的持续运动是唯一例外（见第 6 章）；其余任何元素不做常驻动画。

### 4.1 统一缓动

| 用途 | 曲线 |
| --- | --- |
| 入场 / 显现（元素出现、淡入、浮现） | `cubic-bezier(.16,1,.3,1)`（expo-out） |
| 移动 / 交换（位移、排序、尺寸变化） | `cubic-bezier(.4,0,.2,1)` |
| 弹层（菜单、Tooltip、确认弹窗） | framer-motion spring：`stiffness 380, damping 30` |

CSS 过渡与 framer-motion 动画使用同一组曲线，不得另造第三套缓动。

### 4.2 时长档位

| 档位 | 时长 | 场景 |
| --- | --- | --- |
| 微交互 | 140ms（允许 120–160ms） | hover、按下、勾选、选中态 |
| 常规 | 240ms | 控件状态切换、内容显现、小面板 |
| 页面 / 面板 | 360ms（允许 320–420ms） | 页面切换、账本滑入、抽屉与大面板 |
| 数字滚动 | ≤600ms | 统计读数 count-up，一次性完成，不循环 |

计时仪表与时间之带的专属节奏（逐秒步进、变焦、翻牌、像素点亮）继续由 3.4 与第 6 章定义，不并入上表。

### 4.3 光效边界

- 光色只取自主题 CSS 变量（`--app-accent` / `--app-accent-soft` / `--app-success` / `--app-pause` 等），禁止硬编码色值。
- 形态仅限低透明度（≤0.18）的一次性线性扫光或校准线，随状态转换出现并消退；不使用大面积径向辉光，不做常驻氛围光。
- 禁止：霓虹描边、持续闪烁/呼吸灯、厚重投影、大面积渐变染色、玻璃拟态反光。
- 光效只做强调，不承担信息：移除任何光效后，状态与读数必须仍然完整可读。

### 4.4 reduced-motion 全局降级

- 每个新动效必须同时提供 `prefers-reduced-motion` 降级：改为静态呈现，或仅保留透明度过渡。
- 降级不损失信息：状态、数值与顺序在降级前后一致。
- 运行中切换系统偏好立即生效，无需重启或刷新；时间之带与仪表的具体降级行为见 3.4 与第 6 章。

### 4.5 分页面应用要点

**外壳导航**：导航项选中态用 140ms 微交互过渡（文字/图标颜色、`--app-accent-soft` 底色、底部 2px 短刻度）；页面切换 360ms expo-out，导航轨自身不移动。底部计时状态点仅随状态换色，不闪烁。

**专注页**：状态转换（idle→running→paused→finished）是动效唯一密集区——读数换色 240ms，控制按钮主/次角色互换用标准曲线 240ms 交叉淡入，账本新行 360ms 滑入。idle/finished→running 只播放一次 560ms 纵向校准线横扫，running/paused→finished 只播放一次 720ms 水平校准线收束；两者结束后彻底移除。暂停红只随状态出现，不做提醒式闪烁。

**计时仪表**：五套仪表共享本章缓动；standard/thin/segment 的读数变更只做数值替换或 ≤240ms 过渡，翻牌与像素的机械节奏沿用 3.4 既有参数。样式切换时仪表在 320ms 内完成交接，不闪白、不跳位。

**时间之带**：它是全局唯一持续运动元素，步进/变焦/粒子参数以第 6 章为唯一事实源；周边控制（近景/远景/跟随状态、面板开合）按本章 240/360ms 执行。任何新增光效不得遮挡刻度与标签，透明度 ≤0.18。

**任务页**：行 hover 与「开始专注」渐进显示 140ms；完成勾销与行重排用标准曲线 240ms；撤销 Toast 与确认弹层用 spring（380·30）。任务树展开折叠 240ms，不做级联 stagger。

**统计页**：首次进入分区依次显现（240ms，区段间隔 ≤40ms）；范围切换时图表就地 360ms 过渡，不换页闪动；KPI 与结论读数 count-up ≤600ms，reduced-motion 下直接显示终值。不为图表加辉光或动态渐变。

**设置页**：字体/仪表预览切换 240ms 就地更新；主题切换全窗 360ms 内完成颜色过渡，仅过渡颜色与背景属性，不动布局。仪表预览舞台尺寸稳定，切换不产生二次排版。

**mini 小窗**：收起/展开沿用既有 320ms 收束逻辑，窗口边缘不做光晕；收起态 2px 进度轨仅随秒推进，暂停粒子消散沿用现有 canvas 行为。长任务名往返滚动在 reduced-motion 下退回可聚焦横向滚动（见第 9 章）。

**移动端**：触控反馈 120–140ms，页面与面板 320–360ms；状态确认等待只用按钮内 spinner 或透明度变化，不做骨架屏闪烁。命令等待云端确认期间防重复提交，动效不得暗示操作已成功。

**共享组件**：Tooltip/菜单/弹窗统一 spring（380·30）；Toast 240ms 滑入；Loading/Skeleton 只用透明度脉冲（reduced-motion 下静态）；focus-visible 描边瞬时显示，不做动画。所有组件颜色与光效继续从 token 派生。

## 5. 专注页

信息层级固定为：当前状态 → 当前任务 → 主计时仪表 → 时间之带 → 累计三项 → 控制 → 右侧账本。

- 主计时读数：running = 当前片段时长（专注色），paused = 当前暂停经过（暂停红，继续滴答），idle = 墨色 `00:00`。暂停时读数不降透明度；标签行显示「暂停于 HH:MM · 有效专注 X 已冻结」。
- 普通主窗的仪表舞台宽度固定占满计时区且不随样式切换改变高度；主读数以舞台容器宽度（`cqi`）在 88–160px 间缩放，低高度窗口降为 72–116px，大屏升至 112–190px。不得按整个视口宽度放大，最长读数必须留在仪表边界内。
- 右侧账本与主区同一平面（360px，1600px 以上 420px），1px 发丝竖线分隔，不是浮层。条目 = 时间脊线锚点（HH:MM）+ 序号 + 类型（专注绿/暂停红）+ 时长 + 任务 + 关联/同步语义；进行中条目用浅底强调，时长逐秒刷新。
- 累计专注（绿）/累计暂停（红）/总历时（墨）横排，竖发丝线分隔。
- 账本标题与摘要：running = 当前强调色，paused = 暂停红；正文与分隔仍保持中性，避免整栏染色。
- 控制：暂停 = 墨黑实心，继续 = 当前强调色实心，结束 = 常驻描边。禁止把继续画成暂停红。
- 大窗口（≥1600px）：读数、时间之带、账本全部加档，不允许「细长内容放空」。1280×720 低保档压缩纵向空间但保留全部元素。

### 5.1 五套计时仪表（`timerStyle` 可保存设置）

| 样式 | 构成 |
| --- | --- |
| standard 标准等宽 | JetBrains Mono 固定数字槽 + `ELAPSED / LIVE / HOLD / READY` 工业读数标记，默认 |
| flip 翻页机械 | Oswald + 上下分片卡片 + 中央转轴线；`steady → fold → unfold → commit` 由动画结束事件推进，动画中只保留最新目标；idle/finished 静态提交 |
| pixel 像素点阵 | 自绘 7×9 整数网格数字（`crispEdges`，熄灭格 2.5% 墨），右侧「专注核心」随累计专注点亮（45 分钟充能一整颗，比例可重入） |
| thin 高反差编辑 | Bodoni Moda，衬线粗细对比与编辑排版气质 |
| segment 七段数码 | 自绘 SVG 七段结构，使用真实段码开关，不依赖近似数码字体 |

四套特色仪表与标准读数共享：当前任务、当前状态、累计专注/暂停/总历时、控制与时间之带。

### 5.2 沉浸模式

专注页「全屏沉浸」按钮同时切换 Electron 原生全屏与 body 覆盖层；Esc 键或右上角带 `aria-label` 的退出图标按钮离开，进入与退出不改变计时状态。操作系统拒绝原生全屏时，覆盖层仍提供完整的窗口内沉浸。

- 自上而下固定为：状态与当前任务 → 主计时仪表（五套样式均可用，读数随可用高度放大）→ 累计专注/暂停/总历时与全部控制 → 底部放大的时间之带。
- 时间之带约占 1/3 屏高（36vh，最小 240px），与专注页共享同一 canvas 契约：累计有效专注轴、暂停近景、前沿消散与 reduced-motion 行为完全一致。
- 沉浸中不显示导航轨与右侧账本；覆盖层复用同一 TimerDial 与 TemporalRibbon 实现，不存在第二套仪表代码。
- 进入沉浸的过滤反馈只允许一条 1px 纵向校准线在 620ms 内横扫，不使用宽渐变光带或全屏染色。
- 沉浸覆盖层与普通主窗读取同一个 `data-transition` 状态；用户在沉浸中开始或结束时，一次性扫光/校准线必须出现在覆盖层上，不能只在被遮住的普通工作面播放。
- 普通窗口中的右侧专注账本可折叠，释放水平空间；折叠不改变会话和片段数据。

## 6. 时间之带

时间之带使用本轮真实墙钟作为唯一横轴。专注与暂停都是持续发生的时间段，并与右侧账本的开始/结束时刻对齐；材质由粒子直接构成，不在粒子下方再绘制绿色或红色实心进度条。

- 绿色主体：每段专注以连续可读的绿色粒子时间体保留完整跨度。主体密度有下限，但上下轮廓必须羽化、溢散并带亚像素抖动，不能读成实心塑料条、规则点阵矩形、纤维线或 confetti。
- 生长：运行时绿色粒子体在“现在”前沿持续生成；内部与边缘保持低幅共同流动，历史粒子的世界时间位置稳定，不因区间增长而按百分比重新排布。
- 暂停痕迹：进入 paused 后，红色粒子体从暂停起点持续延伸到“现在”，横向长度必须完整可读。它由低透明度残留层和短寿命活动层组成：残留层保存已流逝暂停时段；活动粒子错峰剥离、漂移、缩小、褪色并死亡，随后以新 cohort 继续表达正在消逝。
- 消散边界：红色越靠近“现在”越密、越旧越散且更接近灰烬色；衰减窗口按绝对秒龄/屏幕距离计算，不随暂停总时长拉伸。允许粒子越过材料床上下边界，但不得形成红色实线、横向连接、实心填充块或无限增长的单粒子对象列表。
- 恢复：继续专注后，已完成的红色暂停残留冻结在历史墙钟位置，不再播放活动 cohort；新的绿色粒子段从其后接续，绿—红—绿边界与账本顺序一致，不回填或抹除真实暂停时间。
- 指针固定在带宽 62% 处，2px 墨色 + 「现在」标签；running 使用绿色状态指针，paused 使用红色状态指针。指针和刻度不能遮挡前沿的破碎边界。
- 墙钟与累计值：墙钟控制横轴、相机、刻度和片段边界；累计有效专注/暂停仍由计时快照控制顶部读数与账本，二者不得互相冒充。
- 近远尺度：秒级近景为 8px/墙钟秒（小格 1 秒、中格 5 秒、大格 1 分钟带时刻标签）；远景为 30 分钟大格约 420px、5 分钟小格。顶部保留「近景 / 远景 / 跟随状态」控制，paused 强制近景，离开暂停后恢复用户先前尺度偏好。
- 变焦只改变同一墙钟坐标系的投影，不销毁或按区间百分比重排历史粒子。尺度变化使用 820ms 对数插值，秒刻度与分钟刻度按密度交接。
- 渲染为单一 canvas；`buildMixedTimelineItems` 提供真实 focus/pause 顺序，`shared/focus/bandMath.ts` 提供确定性 hash、深度和颜色纯函数。历史材料必须可由世界时间坐标重建，不能依赖无限累积的可变粒子数组。
- idle/finished 以最后记录墙钟时刻完全冻结，不再跟随系统时钟漂移；非活动态变焦完成后停止持续重绘，只在尺寸、主题或数据变化时补一帧。
- reduced-motion：保留相同墙钟轴和绿/红完整区间；绿色与红色残留都静态呈现，不播放漂移、缩小、褪色或 cohort 重生，信息和区间长度不得损失。

## 7. 任务页目标

保持现有任务布局与信息结构：左侧任务索引、右侧执行序列/完成档案。

- 连续密排行（无卡片），行高与层级靠缩进与发丝线；当前任务行 = 浅蓝底 + 文字标签，不用左缘竖线。
- 优先级用文字标记（高/中/低），逾期用 warning 色。
- 「开始专注」在行 hover/键盘焦点时渐进显示，不每行常驻大按钮。
- 保持搜索、排序、任务树、关联并开始、6 秒撤销、30/90/365 天完成范围和 120 项逐步加载。

## 8. 统计页目标

统计采用熟悉的顺读日报，不提供抽象可视化切换器：

1. **结论区**：大号有效专注读数 + 一句自然语言结论 + 专注率仪表，明确日期和计算口径。
2. **四项 KPI**：有效专注、暂停损耗、完成轮次/活跃天数、最长一轮；共享一条连续分区，不拆成浮动卡片。
3. **单日时间轴**：上方固定 00/06/12/18/24 的全天定位轨，下方自动放大活跃时段的专注/暂停双车道；旧记录没有片段时显示明确说明，不画空轨。
4. **多日趋势**：使用带 0/中值/最大时长刻度的每日堆叠柱，强调色为专注、红色为暂停；每一天可悬停或键盘聚焦读取精确值，不伪造平滑曲线。
5. **任务投入构成**：100% 构成带 + 精确时长/占比行，最大余数法保证显示百分比合计 100%；已关联、未关联、旧会话汇总分别标记。
6. **暂停损耗**：页尾用专注/暂停守恒轨与每轮平均专注收束结论；最近会话只保留下方唯一账本，Dashboard 内不重复表格。

范围：单日（可前后翻页，不进未来）、近 7 天、半个月、1 个月、自定义。旧请求不得覆盖新结果（request-id 保护保持）。

最近会话账本展开后使用一条连续片段脊线，不嵌套卡片：专注和暂停按真实开始时间交错排列，每行固定包含类型、时长、起止时间与状态标记；专注行另显示任务关联、滴答同步和番茄 To-do 六科归类，暂停行明确说明只计入暂停损耗。六科手动控件始终可见且保持紧凑，未识别时才回落到「学习」。

删除会话必须使用 portal 到 `document.body` 的顶层 `alertdialog`，遮罩覆盖完整视口，危险主按钮使用深红 `btn-danger`，打开后默认聚焦「取消」并锁住 Tab。正文必须点明会话开始时间/有效专注，以及三类后果：FocusLink 本地记录永久删除、已同步滴答记录删除、番茄 To-do 仅清理本机记录且不能声称远端删除已验证。

## 9. 设置与小窗目标

设置页「界面与体验」包含：外观（明亮/深色/跟随系统）、界面字体（六套真实不同骨架）、全局强调色（五选）、计时仪表（五卡实时渲染预览，非截图）、状态色说明、小窗与沉浸偏好。字体和仪表必须使用独立响应式网格；仪表预览舞台尺寸稳定，标准等宽完整显示，不能把宽控件塞回普通设置行造成标签或读数裁切。

设置页「同步」提供默认关闭的「PC 参与实时专注」。开启后必须明确云端是活动计时唯一事实源、断线时控制不可确认，并显示实时连接、revision 与状态；不得继续展示“运行中的计时只归当前设备”。切换事实源只能在本地计时 idle 时生效。实时握手成功前不得把开始动作路由到未确认的云端；服务不可达时普通桌面计时保持本地优先，设置页显示断开状态并提供 [SYNC_TROUBLESHOOTING.md](../backend-design/SYNC_TROUBLESHOOTING.md) 排查入口。

小窗尺寸唯一真值是 `shared/miniWindowLayout.ts`：

- 收起 `184×35`：状态点 + 状态词、当前时间（25px）、2px 进度轨与状态粒子、展开入口。
- 展开 `256×70`：上栏容纳状态、完整任务与窗口操作；下方以主时间/60 格秒轨和三项累计/全套控制组成紧密双区。结束态主读数显示本轮累计专注而非最后一段。
- 不增加第三尺寸，不允许自由缩放；旧设置中的 320×124 由 `snapMiniWindowSize` 自动归一。
- 原生拖拽释放后先吸附再 320ms 收束自动折叠；展开向显示器内部生长并 clamp 到 work area；平面实色 + 1px 边框 + 顶部状态边，禁玻璃/光晕/胶囊。
- 收起后必须保留主进程确认的 side/corner placement：左右边与四角用同一 `184×35` 画布内的纵向状态轨 + 顶/底进度轨组成 L 型边缘签名，角落同时钉住两个轴；顶部/底部继续使用水平边缘轨。纯展示的秒轨、时间和状态都属于原生可拖区，只有实际按钮使用 `no-drag`，不得因透明 canvas 或进度层造成局部拖不动。
- 时间与按钮分属独立网格行，结构上不允许重叠或换行；暂停粒子必须从当前分钟进度边界按真实毫秒相位分批消散，不得固定从轨道最右端喷出或无限重复同一条 CSS 轨迹。长任务名自动往返，reduced-motion 下提供可聚焦横向滚动；字体切换后必须重新测量。

本次专注账本的所有条目都必须保留状态语义：专注条目使用当前全局强调色，暂停条目使用暂停红，至少覆盖左侧状态轨、时间轴刻点、标题与时长。`is-current` 只增加背景强调，不能成为唯一带颜色的条目。

应用图标与托盘图标共用 `F / L` 双织带字标：F 带秒刻痕，L 穿过 F 并承担当前状态色。托盘前景随系统明暗切换，运行使用专注色、暂停使用暂停红；禁止退回通用圆环、番茄或脉搏曲线图标。

## 10. Web / Android 多端专注控制台

Web/PWA 与 Capacitor Android 继续复用同一套 `src/mobile/` React 界面，但产品能力从只读账本升级为
「实时专注控制台 + 已结束会话账本」。它不复制 Electron renderer；PC 显式启用实时控制后，三端共同控制同一场活动会话：

- 首屏优先展示唯一活动会话。状态固定为待机、专注中、已暂停；可填写本次标题并开始，也可暂停、继续和结束。任何写操作等待云端确认期间都必须防重复提交。
- 专注页在手机、平板和 Web 均显示同一真实墙钟时间之带：移动端使用紧凑自适应 canvas，不提供桌面近景/远景切换，但必须保留完整绿—红—绿区间、暂停残留和 reduced-motion 静态表达；画布下方继续显示三项累计时间与控制。
- 待机时优先提供电脑端任务快照选择器：选择任务会带入标题与稳定任务上下文；无快照、任务不在清单中或用户不想关联时，仍可直接填写标题自由开始。任务快照写入 IndexedDB，PC 关闭后继续显示最后一次成功同步的任务。
- 实时状态由云端 revision 作为权威事实。客户端以响应 `serverTime` 为显示基点逐秒外推三时间；断线时明确标记「离线推算」，不得把本机 tick 冒充另一设备已经确认的状态。
- 所有命令携带幂等 command id 与 `expectedRevision`。版本冲突时本次命令不执行，界面显示提交 revision、当前云端 revision 与当前状态，不静默覆盖，也不臆测一定是另一设备造成。
- 实时活动态是服务端权威的 compare-and-swap 寄存器，不做字段级或 CRDT 合并：客户端只接受更高 revision；同 revision 仅允许同一 session/state 的较新时间物化，禁止旧响应或同版本异态倒灌。连接身份切换后，旧 endpoint/token 的在途响应必须失效。未配置令牌时不得恢复或展示旧活动快照；离线缓存必须明确标记「缓存」，不能和已确认云端态并列成叠加态。
- ACK 文案必须区分 applied、duplicate、conflict、rejected：duplicate 表示没有再次执行；conflict 表示本次操作未执行并显示提交 revision 与云端 revision；不得笼统宣称「另一设备覆盖」或「已同步到所有设备」。顶部连接色只表达实时控制连接，账本拉取失败不能把仍然正常的实时连接染成错误态。
- 在线时通过有界长轮询等待 revision 变化；页面隐藏、网络切换、连接参数变化和组件卸载必须取消旧请求，重连不得累加轮询或让旧账号数据回填。
- 结束会话后继续复用完整的 Session / Segment / Pause 账本与 IndexedDB 缓存。关联状态仍使用「已关联 / 未关联」，来源设备不能写成任务来源；缓存命中、实时状态确认和账本拉取是三种不同结果。旧安装或账号切换留下的账本 cursor 被服务判定为 `invalid_cursor` 时，移动端应自动清空旧账号缓存并从头重建一次，不向用户暴露英文协议错误，也不得把旧账号会话与新账号结果叠加。
- 移动端不直接调用 dida CLI 或番茄 To-do 本地桥。结束记录会进入 FocusLink 跨设备账本；只有桌面端先导入权威 completed bundle，再真实执行并确认第三方投递后，才能显示对应的「已同步」或「上传已确认」。PC 未开启实时控制时仍保持本地优先，不得暗示移动端已接管 PC 计时。
- Web/PWA 与 Android 使用与桌面一致的四入口信息架构：专注、任务、统计、设置。手机使用底部四项导航和单列工作面；平板及 Web 从 760px 起使用 64px 左侧导航轨，每个视图独占内容区，宽屏专注页内部再分为计时主区与实时状态侧栏。任务视图提供搜索、清单筛选、选择后返回专注和直接关联开始；统计视图承载完整会话账本；连接配置与 Android 系统入口只出现在设置视图。所有输入和动作至少 44px，文本输入不小于 16px；使用 `100dvh`、safe-area 与 320px 下限，360×800、412×915、820×1180、1440×900 及平板横竖屏不得横向溢出。
- 移动端直接加载 `temporal-foundation.css` 的语义 token、字体和五色强调映射，不再维护独立的硬编码绿色/Inter 主题。设置视图提供与桌面同名的明亮/深色/跟随系统、六套字体和五种强调色选择，并把选择持久化到该设备；响应式布局可以不同，但颜色、字体、焦点环、暂停红和危险深红必须保持同一语义。
- 任务视图按电脑端项目/清单分组，默认只显示至少 52px 高的清单头与任务数量；用户展开后才渲染该清单任务。当前选中任务所在组自动展开，搜索或指定清单筛选时展开匹配组；不得在首屏一次铺开全部清单内容。
- 统计视图直接复用 `shared/sessionAnalytics.ts` 的自然日裁切口径，提供今天/近 7 天/近 30 天三段切换、有效专注结论、专注率、四项 KPI、每日趋势、任务投入 100% 构成、六科/学习投入、24 小时专注/暂停分布、日期热力与暂停守恒收束。热力日期可下钻到按日分组的唯一会话账本，账本再展开 Segment/Pause；页面不得在图表上方和下方重复两套汇总。窄于 370px 的 30 日热力使用 5 列，其他手机 7 列，平板分析区在 900px 起并排，宽屏账本保持唯一且可作为右侧阅读列。
- Web/PWA 访问令牌默认只存 `sessionStorage`；只有用户勾选「在此设备记住令牌」才写入 `localStorage`。Android 活动通知为后台轮询与系统动作把当前令牌另存为 Android Keystore 加密密文，不写明文 SharedPreferences；WebView 会话被系统回收时不得自动清掉仍服务于活动会话的原生密文，只有用户明确移除令牌或切换 endpoint/token/device identity 时才清除。所有端均提供移除令牌与仅清除本机缓存的操作；切换 endpoint 或 token 必须先清空旧账号缓存与实时快照。
- 桌面内置同步服务只绑定 `127.0.0.1`，是 loopback-first 测试后端，不提供 Wi-Fi 局域网直连。Android 使用该地址前必须保持电脑 FocusLink 运行，并通过 USB 执行 `adb reverse tcp:18787 tcp:18787`；手机上的 localhost/127.0.0.1 指手机自身。无线或异地连接只使用 HTTPS 个人云 endpoint；不得建议把内置 HTTP 服务暴露到 `0.0.0.0`，也不得把笼统的“连接失败”作为唯一诊断。
- PWA manifest、页面标题与离线 app shell 使用「FocusLink 专注」语义，不再写「只读预览」。Service Worker 只缓存同源静态资源，不缓存 Bearer 响应或跨源实时接口。
- Capacitor 仅包装同一个 `dist-mobile`，Android 工程只提供薄原生集成。最低 API 24；禁止系统备份 WebView token/账本缓存，Network Security Config 仅为 localhost/127.0.0.1 回环测试开放明文，生产地址必须 HTTPS。
- Android 活动会话使用可见前台通知，动作包含暂停/继续/结束；快捷设置 Tile 在已有活动快照时只转发暂停/继续，待机时打开 App 完成必要输入。原生层不推进业务时钟，也不先行改变状态；通知/Tile 动作进入至少一次命令队列，携带 session/revision。前台 Service 可直接用 Keystore 凭据提交同一个幂等命令，Web 层也可并发重试；只有云端返回 applied/duplicate/conflict/rejected 与最新快照后才移除队列，WebView 被回收不得让系统动作失效。
- Android 把 endpoint、device id 与 token 交给薄原生层，token 必须用 Android Keystore 加密；活动通知 Service 在 WebView 后台时以“主线程延时触发、单线程网络读取、完成后再安排下一轮”的方式每 20 秒读取云端权威快照，网络恢复后自动校准，另一设备结束后移除通知。原生快照 revision 必须单调，idle 也保留云端 revision，WebView 的旧缓存不得覆盖更新状态。系统控制区显示原生连接、省电豁免、后台限制、轮询次数/最近成功/错误，并分别提供省电设置和华为/荣耀/小米自启动设置入口；未取得 OEM 许可时不得宣称绝不会被系统冻结。
- PWA 受浏览器后台冻结与网络调度限制，不能承诺常驻长连接；它必须缓存最后状态、给请求设置硬超时，并在 online/pageshow/重新可见时废弃旧请求立即重连。Android 用前台通知维持活动会话入口，但网络事实仍以云端快照为准。华为、小米等系统可以把标准进行中通知呈现为胶囊、状态栏或左上角入口，具体形态由 OEM 决定，产品文案不得承诺应用可强制开启某一种“灵动岛”样式。
- `cloud/` 当前仍是回环优先测试后端，不具备生产身份、备份、限流与监控；测试通过不等于已上线公共云服务。

## 11. 通用组件与状态

- Tooltip、Toast、菜单、日期选择器、确认弹窗属于统一设计系统（发丝边框、3px 圆角、无玻璃）。Web/Android 的结束专注和清除本机缓存也必须使用可访问的 portal `alertdialog`，支持 Escape、Tab 焦点圈和 reduced-motion；不得回退到原生 `window.confirm`。
- Loading、Skeleton、空数据、首次使用、错误、断线和重试状态明确。
- 滚动条、文本选择（主工作区时间/任务文本可选中复制）、窗口失焦和拖拽行为一致。
- Hover、active、focus-visible、disabled、selected、destructive 各状态完备。
- 关联与同步文案契约：`已关联/未关联` 只表示本地任务关联；`已同步/未同步/同步失败` 只表示滴答云同步队列；番茄 To-do 使用「已写入本地/待上传/上传已确认」。

## 12. 阶段验收

每阶段必须：

1. 提供可运行 Electron 审阅构建，以实际操作手感为主要依据。
2. 提供亮暗状态截图作为排版、溢出和文字清晰度证据。
3. 覆盖 hover、active、focus-visible、disabled、空状态和长文本。
4. 检查默认尺寸、最小尺寸和 1280×720。
5. 运行 format、typecheck、lint、test 和 build。
6. 更新本规范对应章节。
7. 用户明确批准后才进入下一阶段。

阶段 1 截图必须覆盖 idle、running、paused、resume 和 multi-segment；必须验证累计有效专注轴逐秒增长、暂停冻结且保持近景、前沿粒子在有界寿命内消散、恢复无缝续接、页面隐藏恢复与 reduced-motion 静态破碎边缘。
